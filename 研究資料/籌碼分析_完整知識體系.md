# 第一篇：籌碼分析核心理論

## 1.1 籌碼分布原理

### 1.1.1 什麼是「籌碼」？

「籌碼」就是股票。籌碼分析的核心概念是：**每一張股票都有一個買入成本（持倉成本）**，所有持股者在不同價位買入的股票，形成了一個「成本分布圖」。這個分布揭示了市場中多空雙方的力量對比。

**關鍵定義：**
- **籌碼分布（Cost Distribution）**：在某一時間點上，所有流通股按照買入價格繪製的分布圖
- **籌碼密集區（Chip Concentration Zone）**：大量籌碼集中在某一價格區間，形成明顯的「峰」
- **套牢盤（Trapped Longs）**：持倉成本高於當前股價的籌碼
- **獲利盤（Profitable Positions）**：持倉成本低於當前股價的籌碼

### 1.1.2 籌碼密集區 = 支撐／壓力區

**原理：** 當大量籌碼集中在某一價位時，該價位會形成強大的支撐或壓力：

| 位置關係 | 效果 | 心理機制 |
|---------|------|---------|
| 股價在籌碼密集區**上方** | 密集區形成**支撐** | 持有者不願虧損賣出，買盤承接 |
| 股價在籌碼密集區**下方** | 密集區形成**壓力** | 套牢者等待解套，賣壓湧出 |

**判斷標準：**
- 籌碼密集區重合度 > 70%：高度集中，股價波動幅度小
- 籌碼密集區重合度 < 30%：高度分散，股價波動幅度大
- 單一密集區持股量佔流通股 > 20%：該區間有強大支撐/壓力效果

### 1.1.3 籌碼峰的移動與變化含義

| 模式 | 描述 | 市場含義 |
|------|------|---------|
| **低位單峰密集** | 經過長期下跌，籌碼從高位轉移至低位，形成單一密集峰 | 底部訊號，充分換手後主力可能完成建倉 |
| **高位單峰密集** | 經過長期上漲，籌碼從低位轉移至高位 | 頂部警訊，低位獲利盤已出場，後續可能下跌 |
| **雙峰分布** | 籌碼在高低兩個價位各有一個峰 | 多空分歧大，股價在兩峰之間震盪 |
| **多峰分布** | 籌碼分散在多個價位 | 無明確方向，等待突破 |
| **低位密集向上發散** | 籌碼從低位密集區開始向上移動 | 主力拉升，上漲趨勢啟動 |
| **高位密集向下發散** | 籌碼從高位密集區開始向下移動 | 主力出貨，下跌趨勢啟動 |

### 1.1.4 套牢盤與獲利盤的心理與行為

**套牢盤心理：**
1. **淺套（虧損 < 5%）**：持有觀望，期待反彈，賣壓較小
2. **中套（虧損 5-20%）**：心態焦躁，部分認賠出場，逢反彈解套賣出
3. **深套（虧損 > 20%）**：放棄掙扎或死守不賣，形成「死籌碼」，短期賣壓反而減少

**獲利盤行為：**
1. **小幅獲利（< 10%）**：容易落袋為安，尤其散戶
2. **中幅獲利（10-30%）**：分批獲利了結
3. **大幅獲利（> 30%）**：若為主力，可能繼續持有等待更高目標；若為散戶，多已出場

**實際應用：**
- 當籌碼峰從高位快速消失 => 套牢盤認賠，恐慌性殺盤（可能接近底部）
- 當低位籌碼峰不動而價格上漲 => 主力鎖倉，惜售（強勢上漲訊號）
- 當低位籌碼峰大量消失且股價滯漲 => 主力出貨（危險訊號）

## 1.2 主力行為學

### 1.2.1 主力的四個階段

主力操作一檔股票通常經歷**建倉、洗盤、拉升、出貨**四個階段。每個階段有其獨特的量價特徵：

#### 階段一：建倉（吸貨）

**目標：** 在低價位大量買入，收集足夠的籌碼

**時間：** 通常數週到數月，甚至半年以上

**量價特徵：**
| 指標 | 表現 |
|------|------|
| 股價 | 低位區間震盪，波動幅度收窄 |
| 成交量 | 底部溫和放量，偶爾出現脈衝式放量 |
| 換手率 | 從極低（< 1%）逐步提高到 3-5% |
| K線 | 長期橫盤，下跌空間有限（有買盤承接） |

**常見建倉手法：**
1. **打壓建倉**：故意壓低股價，製造恐慌，散戶低價拋售，主力趁機吸貨
2. **橫盤建倉**：長期在狹窄區間震盪，消磨散戶耐心，散戶無聊離場
3. **拉高建倉**：快速拉升一波吸引跟風盤，然後回落繼續吸貨（較少見，通常是急需建倉時使用）

**識別信號：**
- 股價長期下跌後在低位橫盤 > 3個月
- 底部逐漸抬高（更高的低點）
- 成交量在底部呈現規律性的放量-縮量循環
- OBV（On-Balance Volume）在股價橫盤期間持續上升

#### 階段二：洗盤（試盤）

**目標：** 清洗不堅定的散戶籌碼，提高市場平均成本，為拉升減輕壓力

**時間：** 數日到數週

**量價特徵：**
| 指標 | 表現 |
|------|------|
| 股價 | 快速下跌或劇烈震盪，跌破重要均線或支撐位 |
| 成交量 | 下跌時縮量（表明主力未大量賣出） |
| 換手率 | 相對降低 |
| K線 | 出現長上影線、長下影線、十字星 |

**常見洗盤手法：**
1. **壓低洗盤**：快速拉低股價 5-15%，引發散戶止損出局
2. **震盪洗盤**：大幅上下震盪，忽漲忽跌，散戶追高被套、殺低被洗
3. **橫盤洗盤**：長時間橫盤不動，消磨持股者耐心
4. **跌破支撐洗盤**：故意讓股價跌破重要均線（如20日線），引發技術面止損盤
5. **利空洗盤**：配合利空消息打壓，製造恐慌

**洗盤 vs 出貨的區別（關鍵判斷）：**

| 特徵 | 洗盤 | 出貨 |
|------|------|------|
| 下跌時成交量 | **縮量** | **放量** |
| 下跌速度 | 快速猛烈（一兩天完成） | 緩慢陰跌（持續多天） |
| 跌幅 | 通常 < 15%（不破關鍵支撐） | 可以 > 20%（破位下跌） |
| 均線系統 | 中長期均線仍上行 | 中長期均線開始走平或下彎 |
| 盤後籌碼 | 主力持股未明顯減少 | 主力持股持續減少 |
| 目的 | 嚇走散戶，為拉升準備 | 真正賣出股票，結束操作 |

#### 階段三：拉升

**目標：** 快速推高股價，獲取利潤空間

**時間：** 數週到數月

**量價特徵：**
| 指標 | 表現 |
|------|------|
| 股價 | 沿著上升趨勢線或均線穩步上漲 |
| 成交量 | 漲時放量、跌時縮量（健康量價關係） |
| 換手率 | 適中，3-8% |
| K線 | 持續出現中大陽線，回調幅度小、時間短 |

**拉升模式：**
1. **慢牛拉升**：沿20日/60日均線緩步上升，每日漲幅 1-3%，持續時間長
2. **急拉模式**：短期內快速拉升 30-50%，常見於小型股
3. **階梯式拉升**：拉升一波後橫盤整理，再繼續拉升，形成階梯狀

**識別信號：**
- 突破長期橫盤區間且成交量放大
- 股價站上所有主要均線（5/10/20/60/120日線多頭排列）
- MACD 金叉且柱狀體持續放大
- 三大法人（台股）或北向資金（A股）持續買超

#### 階段四：出貨

**目標：** 在高位賣出籌碼，將紙上獲利轉為實際獲利

**時間：** 數週到數月（出貨比建倉難度更大）

**量價特徵：**
| 指標 | 表現 |
|------|------|
| 股價 | 高位震盪，上漲乏力，偶爾假突破 |
| 成交量 | **異常放大**（天量），但股價不再創新高 |
| 換手率 | 大幅提高，> 10% 甚至 > 20% |
| K線 | 出現長上影線、高開低走、烏雲蓋頂 |

**常見出貨手法：**
1. **拉高出貨**：利用利好消息配合漲停板吸引跟風，隔日高開低走出貨
2. **高位震盪出貨**：在高位區間反覆震盪，每次反彈賣出一部分
3. **跳水出貨**：直接大量賣出，不顧價格（急於離場）
4. **漲停板出貨**：在漲停價位掛大量賣單，跟風盤買入接走主力籌碼

**出貨前兆：**
- 成交量創歷史天量但股價漲幅收窄
- 利好消息不斷但股價不漲反跌
- 大戶持股比例連續數週下降
- 融資餘額異常增加（散戶追高）
- 法人由買轉賣（特別是外資）

### 1.2.2 主力控盤度判斷

| 控盤程度 | 主力持股比例（估算） | 市場特徵 |
|---------|-------------------|---------|
| 輕度控盤 | 10-30% | 股價波動受大盤影響大，走勢不獨立 |
| 中度控盤 | 30-50% | 開始出現獨立走勢，有時逆大盤 |
| 高度控盤 | 50-70% | 走勢高度獨立，盤中常見異常買賣盤 |
| 完全控盤 | > 70% | 股價完全由主力說了算，但流動性差 |

**判斷控盤度的方法：**
- 觀察股價在大盤下跌時是否能獨立支撐
- 觀察成交量：控盤度越高，日常成交量越低（主力不活躍交易時）
- 觀察盤口：頻繁出現大買單護盤或大賣單壓盤
- 觀察漲跌停板：控盤度高的股票更容易拉到漲停

## 1.3 市場參與者結構

### 1.3.1 機構投資者 vs 散戶

| 特徵 | 機構投資者 | 散戶 |
|------|-----------|------|
| 資金規模 | 億級以上 | 萬到百萬級 |
| 研究能力 | 專業研究團隊、模型 | 依賴新聞、社群、直覺 |
| 持股週期 | 中長線為主（季度到年） | 短線為主（日到週） |
| 交易行為 | 有計畫、分批進出 | 追漲殺跌、情緒化 |
| 資訊優勢 | 接觸管理層、一手數據 | 依賴公開資訊，滯後 |
| 對股價影響 | 大，可單獨影響走勢 | 小，但群體效應明顯 |

### 1.3.2 聰明錢（Smart Money）vs 笨錢

**聰明錢的特徵：**
- 在恐慌中買入，在狂熱中賣出（逆向操作）
- 提前佈局，在消息公布前已經建倉
- 分批操作，不追漲殺跌
- 代表：大型機構、對沖基金、產業資本、公司內部人

**笨錢的特徵：**
- 追漲殺跌，總是在高點買入、低點賣出
- 受情緒和媒體影響大
- 典型指標：融資餘額暴增（散戶借錢追高）、基金申購量創新高
- 代表：缺乏經驗的散戶投資者

**聰明錢指標（Smart Money Indicator）：**
- 開盤第一個小時的交易方向（笨錢主導，情緒化交易）
- 收盤最後一個小時的交易方向（聰明錢主導，深思熟慮）
- 如果開盤賣、收盤買 => 聰明錢看多
- 如果開盤買、收盤賣 => 聰明錢看空

### 1.3.3 全球主要市場參與者分類

**台股：**
- 外資（約佔成交量 25-35%）
- 投信（約佔 3-5%）
- 自營商（約佔 5-8%）
- 散戶（約佔 50-60%）

**A股：**
- 公募基金（約佔流通市值 8-10%）
- 私募基金
- 北向資金（外資透過滬深港通）
- 社保基金、保險資金
- 散戶（佔比高達 60-70%）

**美股：**
- 機構投資者（佔比 > 70%）
- 對沖基金
- 共同基金、ETF
- 散戶（佔比約 20-25%，2021年後有所上升）
- 做市商（Market Maker）

**港股：**
- 外資機構（佔比 > 40%）
- 南向資金（中國大陸資金）
- 本地機構
- 散戶（佔比約 15-20%）

# 第二篇：台股／A股籌碼分析

## 2.1 三大法人（台股）/ 北向資金（A股）

### 2.1.1 外資 / QFII

**台股外資：**

| 項目 | 說明 |
|------|------|
| 資金規模 | 外資持有台股市值約 40-45%（為最大參與者） |
| 偏好標的 | 大型權值股（台積電、聯發科、鴻海等） |
| 操作風格 | 中長線為主，偏好趨勢交易 |
| 影響力 | 對大盤指數影響最大 |

**外資買賣超判斷標準：**

| 訊號 | 條件 | 意義 |
|------|------|------|
| **強烈看多** | 連續買超 > 5天，且金額遞增 | 中期上漲趨勢確認 |
| **溫和看多** | 間歇性買超，金額不大 | 可能在建倉，觀望中 |
| **強烈看空** | 連續賣超 > 5天，且金額遞增 | 中期下跌風險高 |
| **系統性風險** | 單日賣超 > 300億 | 可能有全球性利空 |

**三大法人同步訊號（台股最強訊號之一）：**
- 三大法人同步買超：**極強看多**，出現的頻率低，但勝率高
- 三大法人同步賣超：**極強看空**，需高度警惕
- 外資買、投信買、自營賣：**偏多**，自營商可能是避險部位

**A股北向資金：**

| 項目 | 說明 |
|------|------|
| 資金規模 | 累計持股市值約 2.5萬億人民幣（2025年底） |
| 偏好標的 | 白酒（貴州茅台）、新能源（寧德時代）、科技（騰訊、小米） |
| 操作風格 | 趨勢跟蹤，配置型為主 |
| 影響力 | 被稱為「聰明錢」的代表 |

**北向資金判斷標準：**

| 訊號 | 條件 | 意義 |
|------|------|------|
| **強力流入** | 單日淨買入 > 100億人民幣 | 外資大幅看多 |
| **持續流入** | 連續 5天以上淨買入 | 趨勢性看多 |
| **大幅流出** | 單日淨賣出 > 100億人民幣 | 短期風險警訊 |
| **持續流出** | 連續 5天以上淨賣出 | 系統性風險上升 |

## 2.2 大戶持股分析

### 2.2.1 千張大戶持股比例

台灣證券交易所每週五公布「集保戶股權分散表」，可查詢不同持股級距的股東人數與持股比例。

**持股級距分類：**
| 級距 | 說明 | 代表身份 |
|------|------|---------|
| 1-5張 | 零股到小額投資 | 散戶 |
| 5-50張 | 小額投資 | 一般投資人 |
| 50-200張 | 中額投資 | 中實戶 |
| 200-400張 | 較大額投資 | 大戶 |
| 400-1000張 | 大額投資 | 大戶/法人 |
| > 1000張 | 超大額投資 | 主力/法人/大股東 |

### 2.2.2 判斷標準

| 千張大戶持股比例 | 含義 | 操作建議 |
|----------------|------|---------|
| **< 40%** | 籌碼分散在散戶手中，股價不易大漲 | 觀望或避開 |
| **40-70%** | 籌碼適度集中，最容易拉抬 | **最佳投資區間** |
| **70-80%** | 高度集中，若大股東看好則續漲 | 注意流動性風險 |
| **> 80%** | 過度集中，流動性極差 | 小心主力操控，進出困難 |

### 2.2.3 籌碼流向判斷

| 情境 | 籌碼流向 | 股價影響 |
|------|---------|---------|
| 千張持股增加 + 50張以下持股減少 | 散戶 → 大戶（籌碼集中） | **易漲難跌** |
| 千張持股減少 + 50張以下持股增加 | 大戶 → 散戶（籌碼分散） | **易跌難漲**（主力出貨訊號） |
| 千張持股增加 + 50張以下持股也增加 | 可能是新資金進場 | 需配合成交量判斷 |
| 400張以上連續3週以上增加 | 主力持續吸貨 | 中期看多 |

**實際案例（台股）：**
某股票千張大戶持股比例從45%上升至65%（歷時3個月），期間股價從50元上漲至85元。籌碼集中度上升是推升股價的主要原因之一。

- 各大券商看盤軟體（如凱基、元大、永豐等）

## 2.3 券商分點追蹤

### 2.3.1 什麼是券商分點？

每個券商在全台各地設有分公司（分點），每個分點每天的買賣紀錄都會被證交所公開。透過追蹤特定分點的買賣行為，可以推測主力的操作方向。

### 2.3.2 分點進出表解讀

證交所每日公布「個股券商進出表」，包含：
- 哪些券商分點買進了某股票
- 買進張數與金額
- 哪些券商分點賣出了某股票
- 賣出張數與金額

**關鍵解讀方式：**

| 觀察重點 | 說明 |
|---------|------|
| **連續性** | 同一分點連續5天以上買超同一檔股票 => 主力在建倉 |
| **金額大小** | 單一分點單日買超 > 500張 => 疑似主力大戶 |
| **多點同買** | 多個分點同時買進同一檔 => 可能是同一主力分散下單 |
| **特定分點** | 外資常用券商分點（如摩根大通、美林、瑞銀等） |

### 2.3.3 主力常用券商分點

**外資常用券商分點：**
| 券商分點 | 特徵 |
|---------|------|
| 美林 | 外資大型機構 |
| 摩根大通 | 外資大型機構 |
| 高盛 | 外資大型機構 |
| 瑞銀 | 外資大型機構 |
| 花旗環球 | 外資大型機構 |
| 台灣摩根士丹利 | 外資大型機構 |

**隔日沖常見券商分點：**
| 券商分點 | 行為模式 |
|---------|---------|
| 凱基-台北 | 當天大量買超拉高股價，隔天賣出（隔日沖） |
| 元大-虎尾 | 知名隔日沖分點 |
| 富邦-建國 | 隔日沖風格 |
| 摩根大通 | 部分有外資隔日沖行為 |

### 2.3.4 隔日沖行為模式

**定義：** 當天大量買進拉高股價（甚至拉到漲停），隔天開盤後立即賣出獲利了結。

**識別方法：**
1. 某分點當日買超佔該股成交量 > 20%
2. 該分點歷史上有高頻率的隔日賣出記錄
3. 股價當天漲幅 > 5% 且該分點為最大買超方

**應對策略：**
- 如果發現隔日沖分點大量買入 => 隔天開盤可能高開但迅速回落
- 避免在隔日沖分點拉升後追高
- 可以利用隔日沖分點的出貨造成的低點逢低買入（如果基本面看好）

**數據來源：**
- [台灣證券交易所 券商進出明細](https://bsr.twse.com.tw/)
- [HiStock 券商分點](https://histock.tw/stock/branch.aspx)
- [玩股網 主力進出](https://www.wantgoo.com/stock/major-investors/broker-buy-sell-rank)

## 2.4 融資融券分析

### 2.4.1 基本概念

| 項目 | 融資（Margin Buying） | 融券（Short Selling） |
|------|---------------------|---------------------|
| 定義 | 向券商借錢買股票 | 向券商借股票賣出 |
| 方向 | 看多（借錢買） | 看空（借券賣） |
| 槓桿 | 自備款 40%，借 60% | 保證金 90% |
| 使用者 | 多為散戶 | 散戶與法人 |
| 風險 | 融資維持率不足會被追繳斷頭 | 軋空風險 |

### 2.4.2 融資餘額變化含義

| 情境 | 融資變化 | 股價變化 | 含義 |
|------|---------|---------|------|
| **健康多頭** | 融資溫和增加 | 穩步上漲 | 散戶跟進但不過熱 |
| **過熱警訊** | 融資暴增 | 快速上漲 | 散戶瘋狂追高，注意回調 |
| **主力出貨** | 融資增加 | **股價不漲或微跌** | **最危險訊號！** 主力賣出，散戶借錢接盤 |
| **洗盤完成** | 融資大幅減少 | 下跌後止穩 | 散戶認賠出場，籌碼清洗乾淨 |
| **底部訊號** | 融資極低 | 低位震盪 | 散戶幾乎全部離場，可能是底部 |

### 2.4.3 融資維持率與斷頭賣壓

**融資維持率公式：**
```
融資維持率 = (股票市值 / 融資金額) x 100%
```

**關鍵閾值：**
| 融資維持率 | 狀態 | 行動 |
|-----------|------|------|
| > 166% | 安全 | 無需擔心 |
| 130-166% | 關注 | 開始注意風險 |
| **< 130%** | **追繳** | **券商發出追繳通知，需補繳差額** |
| 不補繳 | **斷頭** | **券商強制賣出（斷頭）** |

**2025年4月實際案例：**
台股遭遇重大震盪，融資維持率下降至133.8%，低於130%的個股數量高達589檔，形成連環追繳潮。大量斷頭賣壓進一步壓低股價，造成惡性循環。

### 2.4.4 券資比與軋空行情

**券資比公式：**
```
券資比 = (融券餘額 / 融資餘額) x 100%
```

**判斷標準：**
| 券資比 | 含義 | 操作考量 |
|--------|------|---------|
| < 10% | 做空力道極低 | 正常多頭 |
| 10-30% | 有一定做空力道 | 關注 |
| **> 30%** | **做空力道很大** | **可能引發軋空行情** |
| **> 50%** | **極端做空** | **高概率軋空** |

**軋空行情（Short Squeeze）觸發條件：**
1. 券資比 > 30%
2. 股價突然止跌反彈
3. 融券回補買單湧入
4. 股價被快速推高
5. 更多融券被迫回補 => 正反饋循環

**數據來源：**
- [台灣證券交易所 融資融券餘額](https://www.twse.com.tw/zh/trading/margin/mi-margn.html)
- [HiStock 融資融券進出行情](https://histock.tw/stock/three.aspx?m=mg)
- [MacroMicro 融資維持率](https://en.macromicro.me/charts/53117/taiwan-taiex-maintenance-margin)
- [Goodinfo 融資融券統計](https://goodinfo.tw/)

**Python 獲取範例：**

```python

import pandas as pd

def get_margin_trading(date_str):
    """
    取得台股融資融券餘額
    date_str: 格式 'YYYYMMDD'
    """
    url = f"https://www.twse.com.tw/rwd/zh/marginTrading/MI_MARGN?date={date_str}&selectType=ALL&response=json"
    headers = {
        'User-Agent': 'Mozilla/5.0'
    }
    response = requests.get(url, headers=headers)
    data = response.json()

    if data['stat'] == 'OK':
        # 取得個股融資融券明細
        df = pd.DataFrame(data['tables'][1]['data'])
        return df
    return None

def analyze_margin_signal(stock_data):
    """
    分析融資融券訊號
    stock_data: 包含融資餘額、融券餘額、股價的 DataFrame
    """
    signals = []

    # 融資增加但股價不漲 => 主力出貨訊號
    if (stock_data['margin_buy_balance'].pct_change(5) > 0.1).any() and \
       (stock_data['close'].pct_change(5) < 0.02).any():
        signals.append("警告：融資增加但股價滯漲，可能是主力出貨")

    # 融資大幅減少 => 洗盤或底部
    if (stock_data['margin_buy_balance'].pct_change(5) < -0.15).any():
        signals.append("注意：融資大幅減少，可能是洗盤完成或接近底部")

    # 券資比 > 30% => 軋空可能
    stock_data['margin_ratio'] = stock_data['short_sell_balance'] / stock_data['margin_buy_balance']
    if (stock_data['margin_ratio'] > 0.3).any():
        signals.append("機會：券資比超過30%，可能出現軋空行情")

    return signals
```

## 2.5 借券賣出分析

### 2.5.1 基本概念

借券賣出（Securities Borrowing and Lending, SBL）是機構投資者透過券商或集保中心借入股票後賣出的做空方式，與融券不同，借券的額度更大、期限更長、費率更低。

| 比較項目 | 融券 | 借券賣出 |
|---------|------|---------|
| 使用者 | 散戶為主 | 機構法人為主 |
| 額度 | 較小 | 較大（可達數千張） |
| 期限 | 6個月內需回補 | 可展延，最長可達1年以上 |
| 費率 | 固定較高 | 協議費率，通常較低 |
| 市場影響 | 小 | 大（代表機構看空態度） |

### 2.5.2 借券餘額與做空力道

| 訊號 | 條件 | 意義 |
|------|------|------|
| **機構看空** | 借券餘額持續增加 > 5天 | 法人/外資認為股價將下跌 |
| **做空力道加大** | 借券賣出量佔成交量 > 10% | 強烈看空訊號 |
| **空方回補** | 借券餘額持續減少 | 做空者認為跌幅已足夠，可能反彈 |
| **軋空前兆** | 借券餘額極高 + 股價止跌 | 空方可能被迫回補 |

### 2.5.3 借券費率變化

| 費率水準 | 含義 |
|---------|------|
| < 1% | 容易借到的股票，做空成本低 |
| 1-5% | 中等需求 |
| 5-10% | 高需求，借券困難 |
| > 10% | 極度搶手，做空成本極高（可能有軋空風險） |

**數據來源：**
- [台灣證券交易所 借券資訊](https://www.twse.com.tw/zh/trading/margin/twt93u.html)
- [MacroMicro 借券餘額](https://en.macromicro.me/collections/46/tw-stock-relative/164/tw-0050-short-sales)

```python
    data = response.json()

    if data['stat'] == 'OK':
        df = pd.DataFrame(data['data'])
        return df
    return None
```

# 第三篇：美股籌碼分析

## 3.1 13F Filing 分析

### 3.1.1 什麼是 13F Filing？

根據美國 SEC 規定，管理資產超過 **1億美元** 的機構投資者必須在每季結束後 **45天內** 提交 Form 13F，揭露其在美國交易所上市的股票持倉。

**關鍵資訊：**
| 項目 | 說明 |
|------|------|
| 申報對象 | 管理資產 > $100M 的機構 |
| 頻率 | 每季度（Q1截止5/15、Q2截止8/15、Q3截止11/15、Q4截止2/15） |
| 揭露內容 | 美國上市股票多頭持倉、部分期權、可轉債 |
| **不揭露** | 空頭持倉、非美國股票、債券、商品 |
| 申報機構數 | 超過 10,000 家 |

**重要限制：**
- 數據滯後 45天以上（季末到申報截止日）
- 只揭露多頭，不顯示空頭或對沖策略
- 機構可能在申報前已大幅調整持倉

### 3.1.2 追蹤頂級基金持倉

**值得追蹤的頂級基金：**

| 基金 | 管理者 | 風格 | 追蹤價值 |
|------|--------|------|---------|
| Berkshire Hathaway | Warren Buffett | 價值投資 | 長期方向指引 |
| Bridgewater Associates | Ray Dalio | 全球宏觀 | 宏觀趨勢判斷 |
| Renaissance Technologies | Jim Simons | 量化交易 | 量化信號參考 |
| Soros Fund Management | George Soros | 全球宏觀 | 宏觀趨勢 |
| Pershing Square | Bill Ackman | 集中投資 | 事件驅動 |
| Appaloosa Management | David Tepper | 逆向投資 | 市場底部訊號 |
| Third Point | Dan Loeb | 激進投資 | 催化劑事件 |

### 3.1.3 分析方法

**持倉變化分析：**
1. **新建倉位（New Position）**：基金首次買入某股票 => 看好訊號
2. **增持（Add）**：增加現有持倉 => 持續看好
3. **減持（Trim）**：部分賣出 => 降低信心
4. **清倉（Sold Out）**：完全賣出 => 不再看好

**群體行為分析：**
- 當多個頂級基金同時增持某股票 => 強烈看多訊號
- 當多個基金同時減持 => 需要警惕

**實際案例：**
2020年Q3，多個知名基金（包括 Berkshire Hathaway）開始大量建倉蘋果（AAPL），蘋果股價隨後在2021年大幅上漲。通過追蹤13F可以提前發現此趨勢。

```python
import pandas as pd
from edgar import Company

def compare_quarterly_holdings(ticker, quarters=2):
    """
    比較兩個季度的13F持倉變化
    """
    company = Company(ticker)
    filings = company.get_filings(form="13F-HR")[:quarters]

    holdings_list = []
    for f in filings:
        report = f.obj()
        df = report.holdings
        df['quarter'] = f.filing_date
        holdings_list.append(df)

    # 合併比較
    if len(holdings_list) >= 2:
        current = holdings_list[0].set_index('nameOfIssuer')
        previous = holdings_list[1].set_index('nameOfIssuer')

        # 新建倉位
        new_positions = current.index.difference(previous.index)
        print("=== 新建倉位 ===")
        print(current.loc[new_positions])

        # 清倉
        sold_out = previous.index.difference(current.index)
        print("\n=== 清倉 ===")
        print(previous.loc[sold_out])

        # 增減持
        common = current.index.intersection(previous.index)
        changes = pd.DataFrame({
            'current_value': current.loc[common, 'value'],
            'previous_value': previous.loc[common, 'value']
        })
        changes['change_pct'] = (changes['current_value'] - changes['previous_value']) / changes['previous_value'] * 100

        print("\n=== 增持最多 ===")
        print(changes.nlargest(10, 'change_pct'))

        print("\n=== 減持最多 ===")
        print(changes.nsmallest(10, 'change_pct'))

compare_quarterly_holdings("BRK-B")
```

## 3.2 Short Interest（空頭利益）

### 3.2.1 核心指標

| 指標 | 定義 | 計算方式 |
|------|------|---------|
| **Short Interest** | 目前被做空的總股數 | 直接數據 |
| **Short Interest Ratio (SIR)** | 做空比例 | 空頭股數 / 流通股數 x 100% |
| **Days to Cover** | 空頭回補天數 | 空頭股數 / 平均日成交量 |
| **Short Float %** | 空頭佔流通股比 | 空頭股數 / 自由流通股數 x 100% |
| **Borrow Fee Rate** | 借券費率 | 年化百分比 |

### 3.2.2 判斷標準

| 指標 | 低 | 中 | 高 | 極端 |
|------|---|---|---|------|
| Short Interest % | < 5% | 5-10% | 10-20% | > 20% |
| Days to Cover | < 2天 | 2-5天 | 5-10天 | > 10天 |
| Borrow Fee | < 1% | 1-5% | 5-20% | > 20% |

### 3.2.3 Short Squeeze 軋空行情

**定義：** 當大量空頭被迫回補（買回股票），買盤湧入推高股價，進而迫使更多空頭回補，形成正反饋循環。

**觸發條件：**
1. Short Interest > 20%（最好 > 40%）
2. Days to Cover > 5天
3. 股價出現利好催化劑（財報超預期、被收購等）
4. 流通股較少（Small Float）
5. 借券費率飆升

**經典案例：GameStop（GME）2021年1月**

| 時間點 | 事件 | 數據 |
|--------|------|------|
| 2020年底 | Short Interest 佔流通股 | **~140%**（部分股票被多次借出做空） |
| 2021/1/13 | Reddit WallStreetBets 散戶開始集體買入 | 股價 $20 |
| 2021/1/22 | Short Interest 仍高達 113% | 股價突破 $65 |
| 2021/1/27 | 最高點 | **股價觸及 $380**（兩週漲幅 1,500%） |
| 空頭損失 | Melvin Capital 虧損 30%，接受 $2.75B 注資 | 全行業空頭損失 $19.75B |

**教訓：**
- 當 Short Interest > 100% 時，理論上所有空頭不可能同時回補
- 散戶的集體力量在社交媒體時代不可忽視
- 券商限制交易（Robinhood 禁止買入 GME）引發巨大爭議

**數據來源：**
- [Fintel Short Interest](https://fintel.io/ss/us/gme)
- [MarketBeat Short Interest](https://www.marketbeat.com/stocks/NYSE/GME/short-interest/)
- [ORTEX](https://app.ortex.com/) - 即時空頭數據（付費）
- [ShortSqueeze.com](https://shortsqueeze.com/)

## 3.3 Options Flow（期權資金流）

### 3.3.1 核心概念

期權資金流分析是追蹤大額期權交易來推測機構投資者方向判斷的方法。由於期權具有槓桿效應，機構往往透過期權市場表達方向性觀點。

### 3.3.2 關鍵指標

**Put/Call Ratio：**

| 比值 | 含義 | 逆向操作 |
|------|------|---------|
| < 0.7 | 市場極度看多（Call 多） | 可能過熱，注意回調 |
| 0.7 - 1.0 | 正常範圍 | 正常 |
| > 1.0 | 市場偏看空（Put 多） | 可能超賣，逆向看多 |
| > 1.5 | 極度恐慌 | **強烈逆向看多信號** |

**Unusual Options Activity（異常期權活動）：**

識別條件：
| 條件 | 閾值 |
|------|------|
| 交易量 vs 未平倉量 | 當日交易量 > 未平倉量的 2倍 |
| 權利金大小 | 單筆 > $100K |
| 相對平均量 | > 3倍平均每日交易量 |

### 3.3.3 訂單類型分析

| 訂單類型 | 定義 | 信號強度 |
|---------|------|---------|
| **Sweep Order** | 跨多個交易所同時掃貨，急切成交 | **最強**（表示極度看多/看空） |
| **Block Trade** | 單筆大額交易（> 10,000手或 > $200K） | **強** |
| **Split Order** | 拆分成多筆小單分散成交 | **中等** |
| **Golden Sweep** | 在 Ask 價格成交的 Sweep 且到期日較遠 | **極強看多** |

### 3.3.4 Dark Pool（暗池）數據

**定義：** Dark Pool 是私人交易場所，不在公開交易所顯示報價，但成交後會報告。

**關鍵數據：**
- Dark Pool 約佔美股總交易量的 **15%**（2025年）
- 大額交易（Block Trade > 10,000股或 > $200K）視為機構足跡
- Dark Pool 買入量 > 賣出量 => 機構看多

**整合分析：**
- 看漲期權 Sweep + Dark Pool 大量買入 => **極高信心看多**
- 看跌期權 Block + Dark Pool 大量賣出 => **極高信心看空**

**數據來源：**
- [Unusual Whales](https://unusualwhales.com/live-options-flow) - 即時期權流分析
- [FlowAlgo](https://flowalgo.com/) - 期權流和暗池數據
- [OptionStrat Flow](https://optionstrat.com/flow)
- [Barchart Unusual Options](https://www.barchart.com/options/unusual-activity)
- [StockGrid Dark Pools](https://www.stockgrid.io/darkpools)
- CBOE 公布的 Put/Call Ratio

**Python 分析範例：**

```python
import yfinance as yf
import pandas as pd

def get_options_overview(ticker):
    """
    取得期權概覽資訊
    """
    stock = yf.Ticker(ticker)

    # 取得所有到期日
    expirations = stock.options
    print(f"{ticker} 可用到期日: {expirations[:5]}...")

    # 取得最近到期日的期權鏈
    if expirations:
        opt_chain = stock.option_chain(expirations[0])
        calls = opt_chain.calls
        puts = opt_chain.puts

        # 計算 Put/Call 比率
        total_call_volume = calls['volume'].sum()
        total_put_volume = puts['volume'].sum()
        pc_ratio = total_put_volume / total_call_volume if total_call_volume > 0 else 0

        print(f"\n到期日: {expirations[0]}")
        print(f"Call 總交易量: {total_call_volume:,.0f}")
        print(f"Put 總交易量: {total_put_volume:,.0f}")
        print(f"Put/Call Ratio: {pc_ratio:.2f}")

        # 找出異常交易量的期權
        calls['vol_oi_ratio'] = calls['volume'] / calls['openInterest'].replace(0, 1)
        unusual_calls = calls[calls['vol_oi_ratio'] > 2].sort_values('volume', ascending=False)

        puts['vol_oi_ratio'] = puts['volume'] / puts['openInterest'].replace(0, 1)
        unusual_puts = puts[puts['vol_oi_ratio'] > 2].sort_values('volume', ascending=False)

        print(f"\n=== 異常 Call 活動 ===")
        print(unusual_calls[['strike', 'volume', 'openInterest', 'vol_oi_ratio', 'lastPrice']].head(5))

        print(f"\n=== 異常 Put 活動 ===")
        print(unusual_puts[['strike', 'volume', 'openInterest', 'vol_oi_ratio', 'lastPrice']].head(5))

        return calls, puts

calls, puts = get_options_overview("AAPL")
```

## 3.4 Insider Trading（內部人交易）

### 3.4.1 什麼是 Form 4？

根據 SEC 規定，公司內部人（Officer、Director、持股 > 10% 的股東）在買賣自家公司股票時，必須在 **2個工作天內** 提交 Form 4 申報。

**三類申報人：**
1. **Officers（高管）**：CEO、CFO、COO 等
2. **Directors（董事）**：董事會成員
3. **10% Owners（大股東）**：持股超過 10% 的個人或實體

### 3.4.2 交易類型

| 交易代碼 | 類型 | 說明 | 參考價值 |
|---------|------|------|---------|
| **P** | **Open Market Purchase** | 公開市場買入 | **最高**（自掏腰包買入） |
| **S** | **Open Market Sale** | 公開市場賣出 | **高**（但需考慮10b5-1計畫） |
| A | Award/Grant | 股票獎勵 | 低（薪酬計畫） |
| M | Option Exercise | 行使選擇權 | 低（已計畫的行為） |
| G | Gift | 贈與 | 低 |

### 3.4.3 集群內部人買入信號（Cluster Buying）

**定義：** 多個內部人在短期內同時買入公司股票，是最強的看多信號之一。

**判斷標準（根據 MarketTriage）：**
- 3個以上不同的內部人
- 全部是公開市場買入（代碼 P）
- 買入時間在 **10個工作天** 以內
- 每筆買入金額 > **$25,000**

**為什麼 Cluster Buying 有效？**
- 內部人最了解公司狀況
- 用自己的錢買入（非公司獎勵）
- 多人同時買入 = 對公司前景高度一致看好
- 研究顯示，Cluster Buying 後 12個月平均超額收益 > 10%

**賣出信號的解讀：**
- 內部人賣出不一定是壞信號（可能是分散風險、個人需求）
- 但如果多個高管同時大量賣出 => 需要高度警惕
- 注意是否在 10b5-1 計畫下（預先計畫的自動賣出，參考價值較低）

**2025年實際案例：**
- Jamie Dimon（JPMorgan CEO）和 Mark Zuckerberg（Meta CEO）在2025年Q1大量賣出股票，但均為 10b5-1 計畫內的預定交易
- 需區分「計畫性賣出」與「臨時性賣出」

**數據來源：**
- [SEC EDGAR Form 4](https://www.sec.gov/cgi-bin/browse-edgar?action=getcompany&type=4&dateb=&owner=include&count=40)
- [OpenInsider](http://openinsider.com/) - 免費 Form 4 篩選器
- [SecForm4.com](https://www.secform4.com/) - 內部人交易追蹤
- [InsiderFinance](https://www.insiderfinance.io/insider-trades)
- [GuruFocus Insider Trading](https://www.gurufocus.com/insider/summary)
- [MarketTriage](https://markettriage.com/insider-trading-signals) - Cluster Buying 信號

## 4.1 南向／北向資金

### 4.1.1 滬港通與深港通

| **深港通（北向）** | 大陸 → 香港 | 大陸投資者透過深交所買港股 |
| **深港通（南向）** | 香港 → 大陸 | 港股投資者透過港交所買A股 |

> 注意：在港股語境中，「北水」通常指從大陸南下到香港的資金（南向資金），因為從大陸角度是「南下」。

### 4.1.2 南向資金（北水）分析

**南向資金偏好：**
- 偏好高股息股（銀行、電信、公用事業）
- 偏好大型科技股（騰訊、美團、小米）
- 偏好AH股折價股（同一公司A股比H股貴時，買入便宜的H股）

**判斷標準：**

| 訊號 | 條件 | 意義 |
|------|------|------|
| **強力南下** | 單日南向淨買入 > 50億港元 | 大陸資金大舉買入港股 |
| **持續流入** | 連續5天以上淨買入 | 趨勢性看好港股 |
| **大幅流出** | 單日南向淨賣出 > 30億港元 | 短期風險 |

### 4.1.3 額度使用率

每日額度限制：
- 北向（買A股）：每日520億人民幣
- 南向（買港股）：每日420億港元

當額度使用率接近上限時，代表買入意願極強。

## 4.2 CCASS 持倉分析

### 4.2.1 什麼是 CCASS？

CCASS（Central Clearing and Settlement System，中央結算系統）是港交所旗下的證券結算系統。所有在港交所交易的股票，最終都會在 CCASS 中進行結算和託管。

### 4.2.2 CCASS 參與者類型

| 前綴 | 類型 | 說明 |
|------|------|------|
| B | Broker（經紀商） | 證券公司 |
| C | Custodian（託管人） | 銀行等託管機構 |
| P | Pledgee（質押人） | 質押融資 |
| A | Clearing House（清算所） | 如中國結算、HKSCC |

### 4.2.3 分析方法

**追蹤異動：**
1. 觀察特定參與者（如中資券商）的持倉變化
2. 大量股份從多個 Broker 轉移到少數 Custodian => 可能是大額收購前兆
3. 某參與者持倉突然大幅增加 => 可能有大資金進場

**注意事項：**
- CCASS 參與者不一定是股票的受益人（可能只是託管）
- 數據只顯示過去12個月（超過12個月需書面申請，最多7年）
- 需要結合其他信息交叉驗證

**數據來源：**
- [HKEX CCASS Shareholding Search](https://www3.hkexnews.hk/sdw/search/searchsdw.aspx)
- [HKEX Participants Shareholding Reports](https://www.hkex.com.hk/Services/Settlement-and-Depository/Participants-Shareholding-Reports-(For-Listed-Issuers)?sc_lang=en)
- [Webb-site CCASS Analysis](https://webb-database.com/ccass/CCASSnotes.asp) - David Webb 的分析系統

**Python 獲取範例：**

```python
import requests
import pandas as pd
from datetime import datetime, timedelta

def get_ccass_holdings(stock_code, date_str):
    """
    取得港股 CCASS 持倉資料
    stock_code: 港股代碼，例如 '00700'（騰訊）
    date_str: 日期格式 'YYYY/MM/DD'
    """
    url = "https://www3.hkexnews.hk/sdw/search/searchsdw.aspx"

    # 需要處理 ASP.NET 的 ViewState
    session = requests.Session()

    # 先取得頁面獲取 ViewState
    page = session.get(url)
    # 解析 ViewState（需要 BeautifulSoup）
    from bs4 import BeautifulSoup
    soup = BeautifulSoup(page.text, 'html.parser')

    viewstate = soup.find('input', {'name': '__VIEWSTATE'})['value']
    viewstate_gen = soup.find('input', {'name': '__VIEWSTATEGENERATOR'})['value']
    event_validation = soup.find('input', {'name': '__EVENTVALIDATION'})['value']

    # 提交搜尋
    data = {
        '__VIEWSTATE': viewstate,
        '__VIEWSTATEGENERATOR': viewstate_gen,
        '__EVENTVALIDATION': event_validation,
        'txtStockCode': stock_code,
        'txtShareholdingDate': date_str,
        'btnSearch': 'Search'
    }

    result = session.post(url, data=data)

    # 解析結果
    soup = BeautifulSoup(result.text, 'html.parser')
    table = soup.find('table', {'class': 'table'})

    if table:
        df = pd.read_html(str(table))[0]
        return df
    return None

# 使用範例
```

## 4.3 沽空比率

### 4.3.1 基本概念

港股允許做空，每日港交所會公布沽空（做空）數據。

**關鍵指標：**
| 指標 | 定義 |
|------|------|
| 沽空金額 | 當日所有沽空交易的總金額 |
| 沽空比率 | 沽空金額 / 當日總成交金額 x 100% |
| 個股沽空金額 | 某股票當日被沽空的總金額 |

### 4.3.2 判斷標準

| 大盤沽空比率 | 含義 |
|------------|------|
| < 10% | 做空力道低，市場偏多 |
| 10-15% | 正常範圍 |
| 15-20% | 做空力道偏高 |
| > 20% | 市場極度悲觀，可能是逆向買入時機 |

**個股沽空分析：**
- 沽空金額連續增加 + 股價下跌 => 空方主導
- 沽空金額增加 + 股價不跌 => 多方強勁承接（可能是底部）
- 沽空金額突然暴增 => 有人大舉做空，關注原因

**Python 獲取範例：**

```python
import yfinance as yf
import pandas as pd

def analyze_etf_flows(tickers, period='3mo'):
    """
    分析 ETF 資金流向（使用價格和成交量作為代理指標）
    """
    results = {}
    for ticker in tickers:
        etf = yf.Ticker(ticker)
        hist = etf.history(period=period)

        # 計算日資金流（成交量 x 收盤價 x 方向）
        hist['money_flow'] = hist['Volume'] * hist['Close'] * \
                             (hist['Close'] - hist['Open']).apply(lambda x: 1 if x > 0 else -1)

        results[ticker] = {
            'total_money_flow': hist['money_flow'].sum(),
            'avg_daily_volume': hist['Volume'].mean(),
            'price_change': (hist['Close'].iloc[-1] / hist['Close'].iloc[0] - 1) * 100,
            'latest_price': hist['Close'].iloc[-1]
        }

    df = pd.DataFrame(results).T
    return df.sort_values('total_money_flow', ascending=False)

etfs = ['SPY', 'QQQ', 'IWM', 'TLT', 'GLD', 'HYG', 'EEM']
df_flows = analyze_etf_flows(etfs)
print(df_flows)
```

## 6.3 VIX 期限結構

**VIX 基礎判斷：**
| VIX 水準 | 市場狀態 |
|---------|---------|

**VIX 期限結構：**
| 結構 | 定義 | 含義 |
|------|------|------|

- 出現頻率低（通常在市場崩盤時）
- 持續超過1週 => 市場恐慌加劇
- Backwardation 轉回 Contango => 恐慌消退，可能是買入時機

# 第七篇：市場微觀結構理論

## 7.1 核心理論模型

### 7.1.1 Kyle 模型（1985）

**核心思想：** 在批量拍賣市場中，知情交易者（Informed Trader）會策略性地交易，不會一次性揭露所有私人信息。

**模型設定：**
- 三類參與者：知情交易者、噪音交易者（Noise Trader）、做市商
- 做市商只能觀察到總訂單流（無法區分知情與噪音）
- 做市商設定向上傾斜的價格函數作為逆向選擇的保護

**關鍵結論：**
1. **Lambda（λ）**：衡量訂單流對價格的影響（Price Impact）
   - λ 越大 => 市場流動性越差，交易成本越高
2. 知情交易者會分散交易，隱藏在噪音中
3. 價格最終收斂到真實價值

**實際意義：**
- 大額訂單會產生價格衝擊（Price Impact）
- 這就是為什麼機構投資者需要使用算法交易來拆分大單
- Dark Pool 的存在就是為了降低大單的 Price Impact

### 7.1.2 Glosten-Milgrom 模型（1985）

**核心思想：** 做市商面對潛在的知情交易者，透過設定買賣價差（Bid-Ask Spread）來保護自己免受逆向選擇損失。

**模型設定：**
- 做市商掛出買價（Bid）和賣價（Ask）
- 每個交易請求可能來自知情或非知情交易者
- 做市商根據貝葉斯法則不斷更新對股票真實價值的估計

**關鍵結論：**
1. **Bid-Ask Spread 的組成：**
   - 逆向選擇成本（Adverse Selection Cost）：對知情交易者的補償
   - 存貨成本（Inventory Cost）：持有風險的補償
   - 訂單處理成本（Order Processing Cost）：固定成本
2. 知情交易者比例越高 => Spread 越寬
3. 這解釋了為什麼小型股（信息不對稱更嚴重）的 Spread 通常更寬

### 7.1.3 兩個模型的關係

Glosten-Milgrom 模型的均衡在極限情況下會收斂到 Kyle 模型的均衡。兩者都揭示了 **信息不對稱** 是市場微觀結構的核心問題。

## 7.2 訂單流分析（Order Flow Analysis）

### 7.2.1 Level 2 數據解讀

| 數據層級 | 包含資訊 | 用途 |
|---------|---------|------|
| Level 1 | 最佳買賣價及量 | 基本報價 |
| Level 2 | 多檔買賣掛單價格與數量 | 判斷供需分布 |
| Level 3 | 完整訂單簿（交易所內部） | 做市商使用 |

### 7.2.2 Bid-Ask Spread 分析

| Spread 特徵 | 含義 |
|------------|------|
| Spread 極窄 | 高流動性，信息對稱 |
| Spread 突然擴大 | 不確定性增加（如財報前） |
| Spread 持續擴大 | 流動性惡化（風險上升） |

**2025年研究發現：**
- 對高頻交易徵稅導致 Bid-Ask Spread 平均增加 **13%**
- 散戶限價單佔所有訂單約 25.5%，但佔成交量的 18.5%
- 74% 的散戶限價單設定在最佳報價之後（不積極）

### 7.2.3 Price Impact（價格衝擊）

**定義：** 一筆交易對市場價格產生的影響

**研究發現：**
- Price Impact 與 Market Depth 的相關係數為 **-0.75**
- 市場深度下降 => Price Impact 上升 => 流動性惡化

**Price Impact 的估算：**
```
永久性 Price Impact = 最終價格變化（反映信息內容）
暫時性 Price Impact = 交易時的價格偏離（反映流動性需求）
```

- Glosten, L.R. & Milgrom, P.R. (1985). "Bid, Ask, and Transaction Prices in a Specialist Market with Heterogeneously Informed Traders." *Journal of Financial Economics*
- Hasbrouck, J. (2007). *Empirical Market Microstructure*

## 8.2 綜合評分系統

| 維度 | 分數範圍 | 權重 |
|------|---------|------|
| 法人動向 | 0-100 | 25% |
| 籌碼集中度 | 0-100 | 20% |
| 融資融券 | 0-100 | 15% |
| 券商分點 | 0-100 | 15% |
| 連續性 | 0-100 | 15% |
| 技術面確認 | 0-100 | 10% |
| **綜合得分** | **0-100** | **100%** |

**操作建議：**
| 綜合得分 | 建議 |
|---------|------|
| > 80 | **強力買入**，籌碼面高度支持 |
| 60-80 | **買入**，籌碼面偏多 |
| 40-60 | **觀望**，籌碼面中性 |
| 20-40 | **謹慎**，籌碼面偏空 |
| < 20 | **避開或放空**，籌碼面極度不利 |

**Python 綜合評分範例：**

```python
import pandas as pd

class ChipAnalysisScorer:
    """
    籌碼分析綜合評分系統
    """

    def __init__(self):
        self.weights = {
            'institutional': 0.25,
            'concentration': 0.20,
            'margin': 0.15,
            'broker': 0.15,
            'continuity': 0.15,
            'technical': 0.10
        }

    def score_institutional(self, data):
        """
        評估法人動向
        data: dict，包含 foreign_net_buy_days, trust_net_buy_days, dealer_net_buy
        """
        score = 50  # 基礎分

        # 外資連續買超
        if data.get('foreign_net_buy_days', 0) >= 5:
            score += 25
        elif data.get('foreign_net_buy_days', 0) >= 3:
            score += 15
        elif data.get('foreign_net_buy_days', 0) <= -5:
            score -= 25
        elif data.get('foreign_net_buy_days', 0) <= -3:
            score -= 15

        # 投信連續買超
        if data.get('trust_net_buy_days', 0) >= 5:
            score += 15
        elif data.get('trust_net_buy_days', 0) >= 3:
            score += 10
        elif data.get('trust_net_buy_days', 0) <= -5:
            score -= 15

        # 三大法人同步
        if data.get('all_three_buy', False):
            score += 10
        elif data.get('all_three_sell', False):
            score -= 10

        return max(0, min(100, score))

    def score_concentration(self, data):
        """
        評估籌碼集中度
        data: dict，包含 major_holder_pct, major_holder_trend
        """
        score = 50

        pct = data.get('major_holder_pct', 50)
        trend = data.get('major_holder_trend', 0)  # 正數=增加，負數=減少

        # 最佳區間 40-70%
        if 40 <= pct <= 70:
            score += 15
        elif pct > 80:
            score -= 10  # 過度集中
        elif pct < 30:
            score -= 20  # 過度分散

        # 趨勢
        if trend > 0:  # 連續增加的週數
            score += min(trend * 5, 25)
        elif trend < 0:
            score += max(trend * 5, -25)

        return max(0, min(100, score))

    def score_margin(self, data):
        """
        評估融資融券
        data: dict，包含 margin_change, price_change, short_ratio
        """
        score = 50

        margin_chg = data.get('margin_change', 0)  # 融資變化百分比
        price_chg = data.get('price_change', 0)     # 股價變化百分比
        short_ratio = data.get('short_ratio', 0)     # 券資比

        # 融資增加但股價不漲 => 危險
        if margin_chg > 10 and price_chg < 2:
            score -= 30

        # 融資減少且股價止穩 => 洗盤完成
        if margin_chg < -15 and price_chg > -5:
            score += 20

        # 券資比高 => 軋空機會
        if short_ratio > 30:
            score += 15
        elif short_ratio > 50:
            score += 25

        return max(0, min(100, score))

    def score_broker(self, data):
        """
        評估券商分點
        data: dict，包含 key_broker_buy_days, day_trader_pct
        """
        score = 50

        # 關鍵券商連續買超天數
        buy_days = data.get('key_broker_buy_days', 0)
        if buy_days >= 5:
            score += 25
        elif buy_days >= 3:
            score += 15

        # 排除隔日沖影響
        day_trader_pct = data.get('day_trader_pct', 0)
        if day_trader_pct > 30:
            score -= 15  # 隔日沖佔比高，不穩定

        return max(0, min(100, score))

    def score_continuity(self, data):
        """
        評估連續性
        """
        score = 50

        # 各維度的連續性
        days = data.get('consistent_signal_days', 0)
        if days >= 10:
            score += 30
        elif days >= 5:
            score += 20
        elif days >= 3:
            score += 10
        elif days <= -5:
            score -= 20

        return max(0, min(100, score))

    def score_technical(self, data):
        """
        評估技術面確認
        """
        score = 50

        if data.get('above_ma20', False):
            score += 10
        if data.get('above_ma60', False):
            score += 10
        if data.get('macd_golden_cross', False):
            score += 10
        if data.get('volume_increase', False):
            score += 10
        if data.get('breakout', False):
            score += 10

        return max(0, min(100, score))

    def calculate_total_score(self, institutional_data, concentration_data,
                              margin_data, broker_data, continuity_data, technical_data):
        """
        計算綜合得分
        """
        scores = {
            'institutional': self.score_institutional(institutional_data),
            'concentration': self.score_concentration(concentration_data),
            'margin': self.score_margin(margin_data),
            'broker': self.score_broker(broker_data),
            'continuity': self.score_continuity(continuity_data),
            'technical': self.score_technical(technical_data),
        }

        total = sum(scores[k] * self.weights[k] for k in scores)

        # 判斷建議
        if total > 80:
            recommendation = "強力買入"
        elif total > 60:
            recommendation = "買入"
        elif total > 40:
            recommendation = "觀望"
        elif total > 20:
            recommendation = "謹慎"
        else:
            recommendation = "避開"

        return {
            'scores': scores,
            'total_score': round(total, 1),
            'recommendation': recommendation
        }

# 使用範例
scorer = ChipAnalysisScorer()

result = scorer.calculate_total_score(
    institutional_data={
        'foreign_net_buy_days': 7,
        'trust_net_buy_days': 4,
        'all_three_buy': True
    },
    concentration_data={
        'major_holder_pct': 55,
        'major_holder_trend': 3  # 連續3週增加
    },
    margin_data={
        'margin_change': -5,
        'price_change': 3,
        'short_ratio': 15
    },
    broker_data={
        'key_broker_buy_days': 6,
        'day_trader_pct': 10
    },
    continuity_data={
        'consistent_signal_days': 7
    },
    technical_data={
        'above_ma20': True,
        'above_ma60': True,
        'macd_golden_cross': True,
        'volume_increase': True,
        'breakout': True
    }
)

print(f"各維度得分: {result['scores']}")
print(f"綜合得分: {result['total_score']}")
print(f"操作建議: {result['recommendation']}")
```

## 8.3 籌碼分析的局限性

### 8.3.1 數據滯後性

| 數據類型 | 滯後時間 |
|---------|---------|
| 三大法人日報 | 當日盤後（最快） |
| 融資融券日報 | 當日盤後 |
| 集保股權分散 | 每週五（滯後約1週） |
| 13F Filing | 季末後45天（最慢） |
| Insider Trading | 2個工作天內 |

### 8.3.2 假訊號類型

| 假訊號 | 說明 | 防範方法 |
|--------|------|---------|
| **假外資** | 本土資金透過海外帳戶偽裝外資 | 觀察是否與其他外資券商同步 |
| **主力誘多** | 故意買入製造看多氛圍再出貨 | 觀察成交量是否配合，看連續性 |
| **主力誘空** | 故意賣出製造恐慌再吸貨 | 跌時是否縮量（洗盤特徵） |
| **拆單偽裝** | 大單拆成小單規避追蹤 | 觀察多個分點同步行為 |
| **期貨避險** | 自營商避險部位不代表方向 | 區分自行買賣與避險部位 |

### 8.3.3 正確使用籌碼分析的原則

1. **永遠不要只看單日數據** -- 至少觀察5天以上的連續性
2. **多維度交叉驗證** -- 法人+籌碼集中度+融資融券+技術面
3. **基本面是基礎** -- 再好的籌碼面，如果公司基本面爛也沒用
4. **嚴格執行停損** -- 籌碼再漂亮也要設好停損（建議 -7% 到 -10%）
5. **區分市場環境** -- 多頭市場和空頭市場的籌碼面解讀不同
6. **注意流動性** -- 小型股的籌碼面更容易被操控
7. **保持懷疑** -- 主力的目的就是讓你誤判，永遠留有餘地

```python
            'ticker': ticker,
            'short_shares': info.get('sharesShort', 0),
            'short_pct_float': info.get('shortPercentOfFloat', 0),
            'days_to_cover': info.get('shortRatio', 0),
            'short_date': info.get('dateShortInterest', 'N/A')
        }

    def us_get_options_sentiment(self, ticker):
        """取得期權情緒"""
        import yfinance as yf
        stock = yf.Ticker(ticker)

        if not stock.options:
            return None

        chain = stock.option_chain(stock.options[0])
        call_vol = chain.calls['volume'].sum()
        put_vol = chain.puts['volume'].sum()

        return {
            'ticker': ticker,
            'expiration': stock.options[0],
            'call_volume': call_vol,
            'put_volume': put_vol,
            'pc_ratio': put_vol / call_vol if call_vol > 0 else None,
            'sentiment': '看空' if put_vol / call_vol > 1 else '看多' if call_vol > 0 else '中性'
        }

    # ==================== 報告生成 ====================

    def generate_report(self, market, data):
        """
        生成籌碼分析報告
        """
        report = f"""
╔══════════════════════════════════════╗
║     籌碼分析報告 - {market}          ║
║     日期: {datetime.now().strftime('%Y-%m-%d')}               ║
╚══════════════════════════════════════╝

"""
        for key, value in data.items():
            report += f"【{key}】\n"
            if isinstance(value, pd.DataFrame):
                report += value.to_string() + "\n\n"
            elif isinstance(value, dict):
                for k, v in value.items():
                    report += f"  {k}: {v}\n"
                report += "\n"
            else:
                report += f"  {value}\n\n"

        return report
```
