## 一、量化選股策略

### 1.1 多因子模型完整體系

多因子模型是量化選股的核心框架，其理論根基源自 Fama-French 三因子模型（1993）及其五因子擴展（2015），能解釋超過 90% 的分散化投資組合報酬，遠超 CAPM 模型的 70%。

#### 1.1.1 價值因子（Value Factor）

**核心邏輯：** 市場長期低估基本面穩健但定價偏低的資產，買入低估資產可獲得超額報酬。

| 指標 | 計算公式 | 適用場景 | 優勢 | 劣勢 |
|------|----------|----------|------|------|
| **P/E（本益比）** | 股價 / 每股盈餘 | 盈利穩定的成熟企業 | 直觀易懂，資料取得容易 | 盈餘易被會計操控，虧損企業無法使用 |
| **P/B（股價淨值比）** | 股價 / 每股淨資產 | 資產密集型產業（銀行、地產） | 不受短期盈餘波動影響 | 輕資產企業（科技）適用性差 |
| **P/S（股價營收比）** | 股價 / 每股營收 | 尚未獲利的成長型企業 | 營收較難操控，虧損企業也可使用 | 不反映獲利能力 |
| **EV/EBITDA** | 企業價值 / 稅息折舊攤銷前利潤 | 跨國比較、併購估值 | 排除資本結構與稅制差異 | EBITDA 忽略資本支出差異 |
| **FCF Yield（自由現金流收益率）** | 自由現金流 / 市值 | 現金流充沛的企業 | 最能反映真實獲利能力 | 資本支出波動大的企業不穩定 |

**數學原理：**

Fama-French HML（High Minus Low）因子：
```
HML = R(高B/M組合) - R(低B/M組合)
```
歷史上 HML 因子年化溢價約 3-5%，但 2010 年後在美股有所衰減。

**Python 實現框架：**
```python
import pandas as pd
import numpy as np

def value_factor_score(df):
    """
    計算價值因子複合得分
    df 需包含欄位: pe_ratio, pb_ratio, ps_ratio, ev_ebitda, fcf_yield
    """
    # 對每個價值指標取排名百分位（值越低越好，取反向排名）
    for col in ['pe_ratio', 'pb_ratio', 'ps_ratio', 'ev_ebitda']:
        df[f'{col}_rank'] = df[col].rank(pct=True, ascending=True)  # 低值排名靠前

    # FCF Yield 值越高越好
    df['fcf_yield_rank'] = df['fcf_yield'].rank(pct=True, ascending=False)

    # 等權複合得分
    rank_cols = [c for c in df.columns if c.endswith('_rank')]
    df['value_score'] = df[rank_cols].mean(axis=1)

    return df
```

#### 1.1.2 質量因子（Quality Factor）

**核心邏輯：** 高質量企業（高 ROE、低負債、穩定獲利）長期跑贏低質量企業，尤其在市場下跌時表現突出。

| 指標 | 計算方式 | 含義 |
|------|----------|------|
| **ROE（股東權益報酬率）** | 淨利潤 / 股東權益 | 衡量股東資本使用效率 |
| **ROA（總資產報酬率）** | 淨利潤 / 總資產 | 衡量全部資產使用效率 |
| **毛利率穩定性** | 過去 N 年毛利率的標準差 | 盈利品質的穩定程度 |
| **Accruals（應計項目）** | (淨利潤 - 營運現金流) / 總資產 | 盈餘品質，低應計比率更佳 |

**數學原理：**

Fama-French RMW（Robust Minus Weak）因子：
```
RMW = R(高獲利能力組合) - R(低獲利能力組合)
```
獲利能力定義為：(營收 - 銷貨成本 - 利息費用 - SG&A) / 帳面權益

Novy-Marx（2013）發現毛利率 / 總資產 是最強的獲利能力預測指標。

**Accruals 異常（Sloan, 1996）：**
```
Accruals = (ΔCA - ΔCash) - (ΔCL - ΔSTD - ΔTP) - Dep
```
- ΔCA = 流動資產變化、ΔCash = 現金變化
- ΔCL = 流動負債變化、ΔSTD = 短期借款變化
- ΔTP = 應付稅款變化、Dep = 折舊攤銷

高 Accruals 的企業未來報酬傾向較差（盈餘品質低）。

**Python 實現框架：**
```python
def quality_factor_score(df):
    """
    計算質量因子複合得分
    df 需包含: roe, roa, gross_margin_std, accruals_ratio
    """
    # ROE、ROA 越高越好
    df['roe_rank'] = df['roe'].rank(pct=True, ascending=False)
    df['roa_rank'] = df['roa'].rank(pct=True, ascending=False)

    # 毛利率波動越低越好
    df['gm_stability_rank'] = df['gross_margin_std'].rank(pct=True, ascending=True)

    # 應計比率越低越好（盈餘品質越高）
    df['accruals_rank'] = df['accruals_ratio'].rank(pct=True, ascending=True)

    rank_cols = ['roe_rank', 'roa_rank', 'gm_stability_rank', 'accruals_rank']
    df['quality_score'] = df[rank_cols].mean(axis=1)

    return df
```

#### 1.1.3 動量因子（Momentum Factor）

**核心邏輯：** 過去一段時間表現強勢的資產在未來一段時間內傾向持續強勢（動量效應），由 Jegadeesh & Titman（1993）最早系統性驗證。

| 指標 | 計算方式 | 常用窗口 | 特點 |
|------|----------|----------|------|
| **價格動量（3M）** | 過去 3 個月累計報酬 | 短期 | 捕捉短期趨勢，換手率高 |
| **價格動量（6M）** | 過去 6 個月累計報酬 | 中期 | 最常用，風險報酬比最佳 |
| **價格動量（12M-1M）** | 過去 12 個月報酬（排除最近 1 個月） | 長期 | 排除短期反轉效應，經典定義 |
| **盈餘動量（SUE）** | (實際 EPS - 預期 EPS) / 標準差 | 季度 | 盈餘驚喜效應，與價格動量互補 |

**數學原理：**

經典 12-1 動量因子（Carhart, 1997）：
```
MOM = R(過去12個月漲幅前10%股票) - R(過去12個月漲幅後10%股票)
```
排除最近 1 個月是為了避免短期反轉效應（short-term reversal）。

**標準化盈餘驚喜（SUE）：**
```
SUE = (EPS_actual - EPS_expected) / σ(EPS_surprise)
```
正 SUE 代表盈餘超預期，歷史上正 SUE 企業在隨後 60 天內平均超額報酬約 2-4%。

**優勢：**
- 歷史上是最強的因子之一，跨市場、跨資產類別均有效
- 與價值因子負相關，兩者搭配可分散風險

**劣勢：**
- 動量崩潰風險（momentum crash）：市場劇烈反轉時損失巨大（如 2009 年 3 月）
- 高換手率帶來較高交易成本
- 在特定市場環境下（如震盪市）效果不佳

**Python 實現框架：**
```python
def momentum_factor(df, prices):
    """
    計算價格動量與盈餘動量
    prices: 含有日期與收盤價的 DataFrame
    """
    # 12-1 價格動量
    ret_12m = prices.pct_change(252)  # 約12個月交易日
    ret_1m = prices.pct_change(21)    # 約1個月交易日
    mom_12_1 = ret_12m - ret_1m

    # 6個月動量
    mom_6m = prices.pct_change(126)

    # 盈餘動量 SUE
    df['sue'] = (df['eps_actual'] - df['eps_expected']) / df['eps_surprise_std']

    return mom_12_1, mom_6m, df['sue']
```

#### 1.1.4 規模因子（Size Factor）

**核心邏輯：** 小型股因流動性差、資訊不透明、機構覆蓋率低，承擔更多風險，因此長期獲得規模溢價。

**數學原理：**

Fama-French SMB（Small Minus Big）因子：
```
SMB = R(小市值組合) - R(大市值組合)
```
歷史上 SMB 年化溢價約 2-3%，但近年在美股明顯減弱。

**注意事項：**
- 小型股溢價在 A 股市場依然顯著
- 需注意殼價值、流動性陷阱等特殊因素
- 實際操作中大資金難以有效利用規模因子（衝擊成本高）

---

#### 1.1.5 低波動因子（Low Volatility Factor / BAB）

**核心邏輯：** 低 Beta 股票的風險調整後報酬優於高 Beta 股票，這違反了傳統 CAPM 的高風險高報酬預期。Frazzini & Pedersen（2014）提出 BAB（Betting Against Beta）因子。

**數學原理：**

BAB 因子構建：
```
BAB = 1/β_L × R(低Beta組合) - 1/β_H × R(高Beta組合)
```
其中 β_L 和 β_H 分別為低 Beta 和高 Beta 組合的平均 Beta 值。透過槓桿調整使兩端的市場曝險相等。

**為何有效（行為金融解釋）：**
- 槓桿限制（leverage constraint）：投資者無法自由加槓桿，傾向買入高 Beta 股票追求報酬，導致高 Beta 被高估
- 彩票偏好（lottery preference）：散戶偏好高波動「博弈型」股票
- 基準追蹤（benchmark hugging）：基金經理不願偏離基準太多

---

#### 1.1.6 流動性因子（Liquidity Factor）

**核心邏輯：** 流動性差的股票因交易困難、持有風險高，投資者要求流動性溢價。

| 指標 | 計算方式 | 含義 |
|------|----------|------|
| **換手率（Turnover）** | 成交量 / 流通股數 | 交易活躍度 |
| **Amihud 非流動性指標** | \|r_t\| / Volume_t 的均值 | 單位交易量對價格的衝擊 |

**Amihud（2002）非流動性指標：**
```
ILLIQ = (1/D) × Σ(|r_t| / V_t)
```
- r_t = 第 t 日報酬率
- V_t = 第 t 日成交金額
- D = 交易天數

**優劣勢：**
- 優勢：在新興市場（A 股、港股）效果顯著
- 劣勢：大資金難以交易低流動性股票，衝擊成本可能吃掉超額報酬

### 1.2 因子加權方法

| 方法 | 原理 | 適用場景 | 優劣勢 |
|------|------|----------|--------|
| **等權加權** | 每個因子權重相同（1/N） | 因子數量少、無歷史資料時 | 簡單穩健，但忽略因子有效性差異 |
| **IC 加權** | 按因子 IC 值大小分配權重 | 有足夠歷史 IC 資料時 | 動態適應，但 IC 不穩定時可能過擬合 |
| **IC_IR 加權** | 按 IC 的均值 / IC 的標準差分配權重 | 需要兼顧有效性與穩定性時 | 考慮了 IC 的穩定性，更為穩健 |
| **最優化加權** | 基於均值-方差優化求解最優權重 | 因子間相關性複雜時 | 理論上最優，但對輸入參數極度敏感，易過擬合 |

**IC 加權公式：**
```
w_i = IC_i / Σ|IC_j|
```

**IC_IR 加權公式：**
```
w_i = (Mean(IC_i) / Std(IC_i)) / Σ|Mean(IC_j) / Std(IC_j)|
```

**IC 評估基準（業界共識）：**
- IC > 0.02：弱有效因子
- IC > 0.05：良好因子
- IC > 0.10：優秀因子（非常罕見）
- ICIR > 0.4-0.6：高度有效且穩定的因子

**Grinold 基本定律：**
```
Alpha ≈ Volatility × IC × Score
IR ≈ IC × √(Breadth)
```
Breadth = 獨立投資決策的數量。IC 即使只有 0.05，若 Breadth 足夠大（例如在 500 檔股票中選股），也能產生有意義的 IR。

### 1.3 因子中性化處理

**目的：** 消除因子中不想要的風險暴露（如行業偏差、市值偏差），提取純粹的 Alpha。

#### 行業中性化

在每個行業內部進行因子排名，而非全市場排名，避免因子得分被行業特性主導。

**方法一：行業內排名法**
```python
df['factor_neutral'] = df.groupby('industry')['factor_raw'].rank(pct=True)
```

**方法二：回歸殘差法**
```python
import statsmodels.api as sm

# 使用行業啞變數回歸
industry_dummies = pd.get_dummies(df['industry'], drop_first=True)
X = sm.add_constant(industry_dummies)
model = sm.OLS(df['factor_raw'], X).fit()
df['factor_neutral'] = model.resid  # 殘差即為行業中性化因子
```

#### 市值中性化

消除因子與市值的相關性，避免因子策略變成純粹的大/小盤策略。

```python
# 對市值對數取回歸殘差
X = sm.add_constant(np.log(df['market_cap']))
model = sm.OLS(df['factor_raw'], X).fit()
df['factor_size_neutral'] = model.resid
```

#### 雙重中性化（行業 + 市值）

```python
X = pd.concat([
    sm.add_constant(np.log(df['market_cap'])),
    pd.get_dummies(df['industry'], drop_first=True)
], axis=1)
model = sm.OLS(df['factor_raw'], X).fit()
df['factor_double_neutral'] = model.resid
```

### 1.4 因子 IC 分析

**IC（Information Coefficient）** 是衡量因子預測能力的核心指標。

```python
from scipy.stats import spearmanr

def factor_ic_analysis(factor_values, forward_returns, n_periods=60):
    """
    計算因子的 Rank IC 時間序列

    參數:
        factor_values: 每期因子值 (date × stock)
        forward_returns: 每期未來報酬 (date × stock)
        n_periods: 計算期數

    返回:
        ic_series: IC 時間序列
        ic_mean: IC 均值
        ic_std: IC 標準差
        icir: IC 信息比率
    """
    ic_list = []
    for date in factor_values.index:
        fv = factor_values.loc[date].dropna()
        fr = forward_returns.loc[date].dropna()
        common = fv.index.intersection(fr.index)
        if len(common) > 30:
            ic, _ = spearmanr(fv[common], fr[common])
            ic_list.append({'date': date, 'ic': ic})

    ic_df = pd.DataFrame(ic_list).set_index('date')
    ic_mean = ic_df['ic'].mean()
    ic_std = ic_df['ic'].std()
    icir = ic_mean / ic_std if ic_std > 0 else 0

    return ic_df, ic_mean, ic_std, icir
```

**IC 分析報告應包含：**
1. IC 均值與 ICIR
2. IC > 0 的比例（勝率）
3. IC 的時間序列圖（觀察衰退趨勢）
4. 分年度 IC 統計（穩定性檢驗）
5. IC 衰減分析（因子暴露後 N 天的 IC 變化）

### 1.5 因子回測框架

**標準回測流程：**

```
1. 資料準備 → 2. 因子計算 → 3. 因子中性化 → 4. 分組回測 → 5. IC分析 → 6. 績效評估
```

**分組回測法（Quintile/Decile Analysis）：**

```python
def factor_backtest(df, factor_col, return_col, n_groups=5):
    """
    因子分組回測
    將股票按因子值分成 n_groups 組，比較各組報酬差異
    """
    results = {}
    for date, group in df.groupby('date'):
        group['group'] = pd.qcut(group[factor_col], n_groups, labels=False) + 1
        for g in range(1, n_groups + 1):
            mask = group['group'] == g
            avg_ret = group.loc[mask, return_col].mean()
            results.setdefault(g, []).append(avg_ret)

    # 計算各組累計報酬
    group_returns = pd.DataFrame(results)
    group_cumulative = (1 + group_returns).cumprod()

    # 多空組合（做多 Top 組，做空 Bottom 組）
    long_short = group_returns[n_groups] - group_returns[1]

    return group_cumulative, long_short
```

**回測報告應包含：**
- 各組年化報酬率、夏普比率、最大回撤
- 多空組合淨值曲線
- 因子報酬的單調性（Q1 < Q2 < ... < Q5）
- 換手率統計
- 扣除交易成本後的淨報酬

## 二、統計套利策略

### 2.1 配對交易（Pairs Trading）

**核心邏輯：** 找到兩個存在長期均衡關係（協整關係）的資產，當價差偏離均衡時建立多空倉位，等待回歸獲利。這是一種市場中性策略，不依賴市場方向。

#### 2.1.1 協整關係識別

**協整（Cointegration）** 不同於相關性。兩個非平穩的時間序列若存在線性組合使其為平穩序列，則稱它們具有協整關係。

**Engle-Granger 兩步驟法：**

第一步：OLS 回歸求對沖比率
```
Y_t = α + β × X_t + ε_t
```

第二步：對殘差序列 ε_t 進行 ADF 單根檢定
```
Δε_t = γ × ε_{t-1} + Σφ_i × Δε_{t-i} + u_t
```
若 γ 顯著為負（p < 0.05），則 Y 與 X 協整。

**Johansen 檢定：** 更適合多資產情境，可同時檢定多組協整關係。

```python
from statsmodels.tsa.stattools import coint, adfuller
from statsmodels.tsa.vector_ar.vecm import coint_johansen

def find_cointegrated_pairs(price_df, significance=0.05):
    """
    在股票池中尋找協整配對
    price_df: columns = 股票代碼, index = 日期
    """
    n = price_df.shape[1]
    keys = price_df.columns
    pairs = []

    for i in range(n):
        for j in range(i+1, n):
            S1 = price_df[keys[i]]
            S2 = price_df[keys[j]]
            score, pvalue, _ = coint(S1, S2)
            if pvalue < significance:
                pairs.append((keys[i], keys[j], pvalue))

    return sorted(pairs, key=lambda x: x[2])
```

#### 2.1.2 價差序列建模

**對沖比率計算：**
- 靜態法：OLS 回歸斜率
- 動態法：Kalman 濾波器（推薦，可捕捉時變對沖比率）

```python
from pykalman import KalmanFilter

def kalman_hedge_ratio(y, x):
    """
    使用 Kalman 濾波器估計動態對沖比率
    """
    delta = 1e-5
    trans_cov = delta / (1 - delta) * np.eye(2)

    obs_mat = np.vstack([x, np.ones(len(x))]).T[:, np.newaxis, :]

    kf = KalmanFilter(
        n_dim_obs=1,
        n_dim_state=2,
        initial_state_mean=np.zeros(2),
        initial_state_covariance=np.ones((2, 2)),
        transition_matrices=np.eye(2),
        observation_matrices=obs_mat,
        observation_covariance=1.0,
        transition_covariance=trans_cov
    )

    state_means, state_covs = kf.filter(y)
    hedge_ratio = state_means[:, 0]

    return hedge_ratio
```

#### 2.1.3 開倉/平倉信號

**Z-Score 交易信號：**
```
Z_t = (Spread_t - μ_spread) / σ_spread
```
- μ_spread：價差的滾動均值
- σ_spread：價差的滾動標準差

| 信號 | 條件 | 操作 |
|------|------|------|
| 開多空倉 | Z < -2.0 | 做多 Y，做空 X |
| 開空多倉 | Z > +2.0 | 做空 Y，做多 X |
| 平倉 | \|Z\| < 0.5 | 回歸均值，獲利了結 |
| 止損 | \|Z\| > 3.5 | 協整關係可能破裂，止損離場 |

```python
def pairs_trading_signals(spread, lookback=60, entry_z=2.0, exit_z=0.5, stop_z=3.5):
    """
    配對交易信號生成
    """
    rolling_mean = spread.rolling(lookback).mean()
    rolling_std = spread.rolling(lookback).std()
    z_score = (spread - rolling_mean) / rolling_std

    signals = pd.Series(0, index=spread.index)
    position = 0

    for i in range(lookback, len(spread)):
        z = z_score.iloc[i]

        if position == 0:  # 無倉位
            if z < -entry_z:
                position = 1   # 做多價差
                signals.iloc[i] = 1
            elif z > entry_z:
                position = -1  # 做空價差
                signals.iloc[i] = -1
        elif position == 1:  # 持有多倉
            if z > -exit_z or z < -stop_z:
                position = 0
                signals.iloc[i] = 0
            else:
                signals.iloc[i] = 1
        elif position == -1:  # 持有空倉
            if z < exit_z or z > stop_z:
                position = 0
                signals.iloc[i] = 0
            else:
                signals.iloc[i] = -1

    return signals, z_score
```

#### 2.1.4 風控設計

- **止損線設定：** Z-Score 超過 3.5 強制止損（協整破裂風險）
- **配對有效期：** 定期重新檢驗協整關係（建議每季度）
- **最大持倉時間：** 設定時間止損（如 20 個交易日未回歸則平倉）
- **資金分散：** 同時持有多組配對，單一配對不超過總資金 10%
- **行業風險：** 避免同一行業的配對佔比過高

### 2.2 統計套利（Statistical Arbitrage）

#### 2.2.1 PCA 主成分分析降維

**核心邏輯：** 用 PCA 提取股票報酬的共同因子，偏離因子模型預測的股票即為套利機會。

**數學原理：**
```
R = F × β + ε
```
- R：股票報酬矩陣（N×T）
- F：因子報酬矩陣（K×T），由 PCA 提取
- β：因子載荷矩陣（N×K）
- ε：殘差（套利信號來源）

```python
from sklearn.decomposition import PCA

def statistical_arbitrage_pca(returns, n_components=5, lookback=252):
    """
    PCA 統計套利策略
    """
    # 步驟一：PCA 提取共同因子
    pca = PCA(n_components=n_components)
    factors = pca.fit_transform(returns.iloc[-lookback:].T).T

    # 步驟二：計算因子載荷
    loadings = pca.components_.T  # N × K

    # 步驟三：計算殘差
    predicted = returns.iloc[-lookback:].values @ loadings @ loadings.T
    residuals = returns.iloc[-lookback:].values - predicted

    # 步驟四：標準化殘差作為交易信號
    residual_z = (residuals[-1] - residuals.mean(axis=0)) / residuals.std(axis=0)

    return residual_z  # 正值 → 做空，負值 → 做多
```

#### 2.2.2 殘差交易策略

殘差代表無法被共同因子解釋的「特異性報酬」。假設殘差序列為均值回歸的：
- 殘差偏高 → 股價被高估 → 做空
- 殘差偏低 → 股價被低估 → 做多

**關鍵假設：**
- 殘差序列是平穩的（可用 ADF 檢定驗證）
- 因子模型穩定（定期重新估計）
- 足夠多的股票以分散特異風險

### 2.3 均值回歸策略

#### 2.3.1 Ornstein-Uhlenbeck（OU）過程

**數學定義：**
```
dX_t = θ(μ - X_t)dt + σ dW_t
```
- θ：均值回歸速度（mean-reversion speed）
- μ：長期均值
- σ：波動率
- W_t：維納過程（布朗運動）

OU 過程是描述均值回歸行為最經典的連續時間模型。

**離散化估計（AR(1) 模型）：**
```
X_t = a + b × X_{t-1} + ε_t
```
- θ = -ln(b) / Δt
- μ = a / (1 - b)
- σ = std(ε) × √(2θ / (1 - b²))

```python
def estimate_ou_parameters(spread, dt=1/252):
    """
    估計 OU 過程參數
    """
    X = spread.values[:-1]
    Y = spread.values[1:]

    # AR(1) 回歸
    b = np.polyfit(X, Y, 1)
    a, slope = b[1], b[0]  # 注意 polyfit 返回順序

    residuals = Y - (a + slope * X)

    theta = -np.log(slope) / dt
    mu = a / (1 - slope)
    sigma = np.std(residuals) * np.sqrt(2 * theta / (1 - slope**2))

    return theta, mu, sigma
```

#### 2.3.2 半衰期計算

**半衰期（Half-life）** 是價差回歸到均值一半距離所需的時間，直接影響交易頻率與持倉週期。

```
Half-life = ln(2) / θ ≈ 0.693 / θ
```

**實用意義：**
- 半衰期 < 5 天：非常快速的均值回歸，適合日內/短線
- 半衰期 5-20 天：最佳交易窗口，適合波段操作
- 半衰期 20-60 天：回歸較慢，需較大耐心與資金
- 半衰期 > 60 天：太慢，不適合交易

```python
def half_life(spread):
    """
    計算均值回歸半衰期
    """
    lag = spread.shift(1).dropna()
    delta = spread.diff().dropna()

    # 回歸: Δy = α + β × y_{t-1} + ε
    X = sm.add_constant(lag)
    model = sm.OLS(delta, X).fit()

    beta = model.params.iloc[1]
    hl = -np.log(2) / beta if beta < 0 else np.inf

    return hl
```

#### 2.3.3 Z-Score 交易信號

```python
def mean_reversion_signals(spread, half_life_val):
    """
    基於半衰期的均值回歸交易信號
    """
    lookback = int(round(half_life_val))
    lookback = max(lookback, 5)  # 至少5天

    z_score = (spread - spread.rolling(lookback).mean()) / spread.rolling(lookback).std()

    signals = pd.Series(0, index=spread.index)
    signals[z_score < -1.5] = 1    # 做多
    signals[z_score > 1.5] = -1    # 做空
    signals[abs(z_score) < 0.5] = 0  # 平倉

    return signals, z_score
```

**統計套利策略優劣勢總結：**

| 面向 | 優勢 | 劣勢 |
|------|------|------|
| 市場曝險 | 市場中性，不依賴方向 | 在極端行情下中性可能失效 |
| 報酬特徵 | 穩定、低波動、夏普比率高 | 單次報酬較小，依賴頻率 |
| 容量 | 可在多市場多配對同時操作 | 大資金進出影響價差 |
| 模型風險 | 有堅實的統計理論基礎 | 歷史關係可能結構性破裂 |

## 三、趨勢跟蹤策略（CTA / Managed Futures）

### 3.1 策略概述

CTA（Commodity Trading Advisor）策略主要在期貨市場運行，核心思想是「趨勢會持續」。AQR 的研究指出，趨勢跟蹤在過去 100 年的大部分危機中都提供了正報酬，是優秀的尾部風險對沖工具。

**CTA 策略的關鍵特徵：**
- 多空皆可操作，不依賴市場漲跌
- 跨資產類別（股指、債券、外匯、商品）
- 系統化規則驅動，排除人為情緒干擾
- 在市場劇烈波動時表現最佳（正凸性）

---

### 3.2 移動平均線交叉策略

**核心邏輯：** 短期均線上穿長期均線（黃金交叉）做多，下穿（死亡交叉）做空。

**經典參數組合：**
- 快線/慢線：10/50、20/100、50/200
- 時間框架：日線為主，週線為輔

**數學表達：**
```
Signal = Sign(MA_fast - MA_slow)
Position = +1 if MA_fast > MA_slow else -1
```

**指數移動平均線（EMA）改良：**
```
EMA_t = α × Price_t + (1 - α) × EMA_{t-1}
α = 2 / (N + 1)
```
EMA 對近期價格給予更多權重，反應更靈敏。

```python
def ma_crossover_strategy(prices, fast=50, slow=200):
    """
    移動平均線交叉策略
    """
    ma_fast = prices.rolling(fast).mean()
    ma_slow = prices.rolling(slow).mean()

    signal = np.where(ma_fast > ma_slow, 1, -1)

    # 計算報酬
    returns = prices.pct_change()
    strategy_returns = signal[:-1] * returns.values[1:]  # 信號延遲一天執行

    return pd.Series(strategy_returns, index=prices.index[1:])
```

**優勢：** 簡單直觀，在強趨勢市中表現優異
**劣勢：** 震盪市頻繁假突破，產生大量交易成本（「鞭尾效應」whipsaw）

---

### 3.3 通道突破策略

#### 3.3.1 Donchian Channel（唐奇安通道）

**核心邏輯：** 價格突破 N 天最高價做多，跌破 N 天最低價做空。源自著名的「海龜交易法則」。

```
Upper = max(High_{t-1}, ..., High_{t-N})
Lower = min(Low_{t-1}, ..., Low_{t-N})
```
- 突破上軌 → 做多
- 跌破下軌 → 做空

經典參數：N = 20（短期）或 55（中長期）

```python
def donchian_breakout(prices_high, prices_low, prices_close, n=20):
    """
    唐奇安通道突破策略
    """
    upper = prices_high.rolling(n).max().shift(1)
    lower = prices_low.rolling(n).min().shift(1)

    signals = pd.Series(0, index=prices_close.index)
    position = 0

    for i in range(n, len(prices_close)):
        if prices_close.iloc[i] > upper.iloc[i] and position <= 0:
            position = 1
        elif prices_close.iloc[i] < lower.iloc[i] and position >= 0:
            position = -1
        signals.iloc[i] = position

    return signals
```

#### 3.3.2 Bollinger Band（布林通道）

```
Middle = MA(N)
Upper = MA(N) + k × σ(N)
Lower = MA(N) - k × σ(N)
```
- 常用參數：N=20, k=2
- 突破上軌可能代表趨勢啟動或超買
- 趨勢策略：價格收在布林上軌之外 → 做多（順勢）
- 均值回歸策略：價格觸及上軌 → 做空（逆勢）

### 3.4 動量策略

#### 3.4.1 時間序列動量（Time-Series Momentum）

**核心邏輯：** 根據資產自身過去的報酬決定多空方向。Moskowitz, Ooi & Pedersen（2012）的研究在 58 種期貨上驗證了時序動量的有效性。

```
Signal_t = Sign(R_{t-1, t-K})
```
若過去 K 個月報酬為正 → 做多，為負 → 做空。

**常用窗口：** 1M、3M、6M、12M，多窗口組合更穩健。

#### 3.4.2 截面動量（Cross-Sectional Momentum）

**核心邏輯：** 在一組資產中，做多相對強勢的，做空相對弱勢的。

```
做多：過去 K 個月報酬排名前 20% 的資產
做空：過去 K 個月報酬排名後 20% 的資產
```

**兩者的比較：**

| 面向 | 時間序列動量 | 截面動量 |
|------|------------|---------|
| 基準 | 資產自身歷史 | 資產間相對排名 |
| 淨曝險 | 可能有方向性曝險 | 天然多空平衡 |
| 適用場景 | 期貨（CTA）常用 | 股票、ETF 常用 |
| 容量 | 較大（單邊操作） | 中等（需要做空） |

---

### 3.5 趨勢強度指標（ADX）

**ADX（Average Directional Index）** 由 Welles Wilder 於 1978 年提出，衡量趨勢的強度而非方向。

**計算步驟：**
```
1. +DM = High_t - High_{t-1} （若為正且 > -DM，否則為 0）
2. -DM = Low_{t-1} - Low_t （若為正且 > +DM，否則為 0）
3. +DI = 100 × EMA(+DM, N) / ATR(N)
4. -DI = 100 × EMA(-DM, N) / ATR(N)
5. DX = 100 × |+DI - -DI| / (+DI + -DI)
6. ADX = EMA(DX, N)
```

**ADX 應用規則：**
- ADX > 25：趨勢明確，適合使用趨勢跟蹤策略
- ADX < 20：市場震盪，應使用均值回歸策略或觀望
- ADX 上升：趨勢增強中
- ADX 下降：趨勢減弱中

**結合 ADX 的策略過濾器：**
```python
def adx_filter(high, low, close, n=14, threshold=25):
    """
    ADX 趨勢過濾器
    只在 ADX > threshold 時啟動趨勢策略
    """
    # 簡化計算（完整計算需 TA-Lib 或 pandas_ta）
    import pandas_ta as ta

    adx_df = ta.adx(high, low, close, length=n)
    adx_values = adx_df[f'ADX_{n}']

    trend_active = adx_values > threshold
    return trend_active, adx_values
```

### 3.6 倉位管理：ATR-Based Position Sizing

**核心邏輯：** 使用 ATR（平均真實波幅）作為波動率代理指標，使每個交易品種對投資組合貢獻相等的風險。

**ATR 計算：**
```
TR_t = max(High_t - Low_t, |High_t - Close_{t-1}|, |Low_t - Close_{t-1}|)
ATR = EMA(TR, N)  # 常用 N=14 或 20
```

**ATR-Based 倉位大小：**
```
Position Size = (Account × Risk%) / (ATR × Multiplier)
```
- Account：帳戶總資金
- Risk%：單一品種目標風險（通常 0.5%-2%）
- ATR：當前 ATR 值
- Multiplier：合約乘數（期貨）

**範例：**
```
帳戶 = 1,000,000
Risk% = 1%
ATR = 50 點
合約乘數 = 10
Position Size = 1,000,000 × 1% / (50 × 10) = 20 口
```

```python
def atr_position_sizing(account_value, atr, multiplier=1, risk_pct=0.01):
    """
    ATR 倉位計算
    """
    dollar_risk = account_value * risk_pct
    position_size = dollar_risk / (atr * multiplier)
    return int(position_size)
```

### 3.7 多品種多時間框架組合

**組合原則：**
1. **資產類別分散：** 股指 + 債券 + 外匯 + 商品（至少 4 類）
2. **品種分散：** 每類至少 3-5 個品種
3. **時間框架分散：** 短期（20日）+ 中期（60日）+ 長期（200日）
4. **策略分散：** 趨勢 + 動量 + 突破 混合

**組合權重分配：**
```
每品種風險預算 = 總風險預算 / 品種數量
每品種倉位 = 風險預算 / (品種波動率 × 合約價值)
```

**CTA 策略優劣勢總結：**

| 優勢 | 劣勢 |
|------|------|
| 危機時正報酬（正凸性） | 震盪市持續虧損 |
| 多空靈活，不依賴方向 | 勝率通常只有 35-45% |
| 跨資產分散風險 | 需要較高耐心與紀律 |
| 規則明確，可完全自動化 | 趨勢晚進晚出，錯過最佳進場 |
| 歷史長期有效（100年+） | 近年部分品種趨勢減弱 |

## 四、機器學習在投資中的應用

### 4.1 監督學習

#### 4.1.1 隨機森林（Random Forest）

**核心邏輯：** 建構多棵決策樹，以投票（分類）或平均（回歸）方式做出預測，天然具有防過擬合的能力。

**在量化投資中的應用：**
- 預測股票未來 N 天漲跌方向（分類問題）
- 預測報酬率大小（回歸問題）
- 因子重要性排名

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import TimeSeriesSplit

def rf_stock_prediction(features, labels, n_estimators=500):
    """
    隨機森林股票預測
    features: 技術指標 + 基本面 + 另類數據
    labels: 未來 N 天漲跌（1/0）
    """
    tscv = TimeSeriesSplit(n_splits=5)

    model = RandomForestClassifier(
        n_estimators=n_estimators,
        max_depth=8,
        min_samples_leaf=50,
        max_features='sqrt',
        random_state=42,
        n_jobs=-1
    )

    scores = []
    for train_idx, test_idx in tscv.split(features):
        X_train, X_test = features.iloc[train_idx], features.iloc[test_idx]
        y_train, y_test = labels.iloc[train_idx], labels.iloc[test_idx]

        model.fit(X_train, y_train)
        scores.append(model.score(X_test, y_test))

    # 特徵重要性
    model.fit(features, labels)
    importance = pd.Series(
        model.feature_importances_,
        index=features.columns
    ).sort_values(ascending=False)

    return model, scores, importance
```

#### 4.1.2 XGBoost / LightGBM

| 適用場景 | 中小規模數據 | 大規模、高維度數據 |

### 4.2 深度學習

#### 4.2.1 LSTM（長短期記憶網路）

**核心邏輯：** LSTM 是 RNN 的改良版本，透過記憶門（forget gate）、輸入門（input gate）、輸出門（output gate）解決長期依賴問題，特別適合金融時間序列。

**數學原理：**
```
遺忘門: f_t = σ(W_f · [h_{t-1}, x_t] + b_f)
輸入門: i_t = σ(W_i · [h_{t-1}, x_t] + b_i)
候選值: C̃_t = tanh(W_c · [h_{t-1}, x_t] + b_c)
記憶更新: C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t
輸出門: o_t = σ(W_o · [h_{t-1}, x_t] + b_o)
隱藏狀態: h_t = o_t ⊙ tanh(C_t)
```

**最新研究表明：** LSTM 達到最高的峰值召回率（73.4%），預測曲線與實際市場趨勢密切匹配。

```python
import torch
import torch.nn as nn

class LSTMPredictor(nn.Module):
    """
    LSTM 股價預測模型
    """
    def __init__(self, input_size, hidden_size=64, num_layers=2, dropout=0.2):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            dropout=dropout,
            batch_first=True
        )
        self.fc = nn.Sequential(
            nn.Linear(hidden_size, 32),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(32, 1)
        )

    def forward(self, x):
        # x shape: (batch, seq_len, features)
        lstm_out, _ = self.lstm(x)
        # 取最後一個時間步的輸出
        last_hidden = lstm_out[:, -1, :]
        output = self.fc(last_hidden)
        return output

# 訓練範例
def train_lstm(model, train_loader, val_loader, epochs=100, lr=0.001):
    optimizer = torch.optim.Adam(model.parameters(), lr=lr)
    criterion = nn.MSELoss()
    scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(optimizer, patience=10)

    best_val_loss = float('inf')
    for epoch in range(epochs):
        model.train()
        for X_batch, y_batch in train_loader:
            optimizer.zero_grad()
            pred = model(X_batch)
            loss = criterion(pred.squeeze(), y_batch)
            loss.backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
            optimizer.step()

        # 驗證
        model.eval()
        val_loss = 0
        with torch.no_grad():
            for X_val, y_val in val_loader:
                pred = model(X_val)
                val_loss += criterion(pred.squeeze(), y_val).item()

        scheduler.step(val_loss)
        if val_loss < best_val_loss:
            best_val_loss = val_loss
            torch.save(model.state_dict(), 'best_model.pth')

    return model
```

#### 4.2.2 Transformer 用於時序預測

**核心邏輯：** Transformer 的自注意力機制可平行處理整個序列，捕捉任意距離的依賴關係，在 NLP 革命後也被廣泛應用於金融時序。

```python
class TransformerPredictor(nn.Module):
    """
    Transformer 金融時序預測模型
    """
    def __init__(self, input_size, d_model=64, nhead=4, num_layers=2, dropout=0.1):
        super().__init__()
        self.input_proj = nn.Linear(input_size, d_model)
        self.pos_encoding = nn.Parameter(torch.randn(1, 500, d_model))

        encoder_layer = nn.TransformerEncoderLayer(
            d_model=d_model,
            nhead=nhead,
            dim_feedforward=d_model * 4,
            dropout=dropout,
            batch_first=True
        )
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=num_layers)
        self.fc_out = nn.Linear(d_model, 1)

    def forward(self, x):
        # x shape: (batch, seq_len, features)
        x = self.input_proj(x) + self.pos_encoding[:, :x.size(1), :]
        x = self.transformer(x)
        x = x[:, -1, :]  # 取最後時間步
        return self.fc_out(x)
```

```python
            self.features[self.current_step],
```

### 4.5 過擬合防範

**過擬合是機器學習在金融領域最大的敵人。** 金融數據信噪比極低（Signal-to-Noise Ratio），模型很容易學到噪音而非真正的規律。

#### 4.5.1 時間序列交叉驗證

**絕對不可使用標準 K-Fold 交叉驗證！** 因為金融數據有時間順序性，未來數據不能出現在訓練集中。

```python
from sklearn.model_selection import TimeSeriesSplit

def walk_forward_validation(model, X, y, n_splits=5):
    """
    Walk-Forward 驗證
    訓練集只包含過去的數據，測試集是未來一段時間
    """
    tscv = TimeSeriesSplit(n_splits=n_splits)
    results = []

    for fold, (train_idx, test_idx) in enumerate(tscv.split(X)):
        X_train, X_test = X.iloc[train_idx], X.iloc[test_idx]
        y_train, y_test = y.iloc[train_idx], y.iloc[test_idx]

        model.fit(X_train, y_train)
        pred = model.predict(X_test)

        # 評估
        ic = spearmanr(pred, y_test)[0]
        results.append({
            'fold': fold,
            'train_size': len(train_idx),
            'test_size': len(test_idx),
            'ic': ic,
            'train_end': X.index[train_idx[-1]],
            'test_start': X.index[test_idx[0]],
            'test_end': X.index[test_idx[-1]]
        })

    return pd.DataFrame(results)
```

### 4.6 模型評估：準確率 vs 盈利能力

**關鍵洞察：準確率高不代表能賺錢，能賺錢不代表準確率高。**

| 指標 | 含義 | 與盈利的關係 |
|------|------|-------------|
| 準確率（Accuracy） | 預測方向正確的比例 | 不考慮盈虧金額大小 |
| 精確度（Precision） | 預測為漲中真正漲的比例 | 高精確度減少假信號損失 |
| 召回率（Recall） | 真正漲的中被預測到的比例 | 高召回率抓住更多獲利機會 |
| **IC（Rank IC）** | 預測值與實際報酬的排名相關 | **最貼近實際投資績效** |
| **盈利因子（Profit Factor）** | 總獲利 / 總虧損 | 直接反映策略盈利能力 |
| **夏普比率** | 風險調整後報酬 | 綜合考慮報酬與風險 |

**建議的評估優先順序：**
1. 夏普比率 / Calmar Ratio（首要）
2. IC / ICIR（因子模型）
3. 最大回撤（風險控制）
4. 盈利因子（交易策略）
5. 準確率（僅作參考）

### 5.2 滑點與衝擊成本建模

**交易成本的組成：**
```
總交易成本 = 佣金 + 買賣價差 + 市場衝擊 + 時機成本
```

**市場衝擊模型（Almgren-Chriss, 2001）：**

臨時衝擊（Temporary Impact）：
```
g(v) = η × v  （線性模型）
g(v) = η × v^δ  （冪律模型, δ ≈ 0.5-0.6）
```
- v = 交易速度（每單位時間的交易量）
- η = 衝擊係數

永久衝擊（Permanent Impact）：
```
h(v) = γ × v
```

**平方根衝擊模型（業界最常用）：**
```
Impact = σ × √(Q / V)
```
- σ = 日波動率
- Q = 交易量
- V = 日均成交量

```python
def estimate_market_impact(order_size, daily_volume, daily_volatility, impact_coeff=0.1):
    """
    估算市場衝擊成本
    """
    participation_rate = order_size / daily_volume
    impact = impact_coeff * daily_volatility * np.sqrt(participation_rate)
    impact_cost = impact * order_size
    return impact, impact_cost
```

### 5.3 TWAP 與 VWAP 算法

#### TWAP（Time-Weighted Average Price）

**核心邏輯：** 將大單均勻分割成小單，在指定時間窗口內等時間間隔執行。

```python
def twap_schedule(total_quantity, duration_minutes, interval_minutes=5):
    """
    TWAP 拆單計畫
    加入隨機化以防止被偵測
    """
    n_slices = duration_minutes // interval_minutes
    base_qty = total_quantity // n_slices
    remainder = total_quantity % n_slices

    schedule = []
    for i in range(n_slices):
        qty = base_qty + (1 if i < remainder else 0)
        # 加入 ±20% 隨機化
        randomized_qty = int(qty * np.random.uniform(0.8, 1.2))
        # 時間也加入隨機偏移
        time_offset = np.random.uniform(-30, 30)  # 秒
        schedule.append({
            'slice': i,
            'quantity': randomized_qty,
            'scheduled_time': i * interval_minutes,
            'time_offset_sec': time_offset
        })

    # 調整使總量正確
    total_scheduled = sum(s['quantity'] for s in schedule)
    schedule[-1]['quantity'] += total_quantity - total_scheduled

    return schedule
```

#### VWAP（Volume-Weighted Average Price）

**核心邏輯：** 根據歷史成交量的日內分布模式分配交易量，在成交量大的時段多交易、成交量小的時段少交易。

**VWAP 是最廣泛使用的執行基準。**

```python
def vwap_schedule(total_quantity, volume_profile):
    """
    VWAP 拆單計畫
    volume_profile: 日內各時段的歷史平均成交量佔比
    """
    schedule = []
    for period, vol_pct in enumerate(volume_profile):
        qty = int(total_quantity * vol_pct)
        schedule.append({
            'period': period,
            'target_quantity': qty,
            'volume_participation': vol_pct
        })

    # 確保總量正確
    total_scheduled = sum(s['target_quantity'] for s in schedule)
    schedule[-1]['target_quantity'] += total_quantity - total_scheduled

    return schedule
```

**TWAP vs VWAP 比較：**

| 面向 | TWAP | VWAP |
|------|------|------|
| 複雜度 | 低 | 中（需成交量預測） |
| 市場衝擊 | 中（不考慮流動性） | 低（順應流動性） |
| 可預測性 | 高（容易被偵測） | 中 |
| 適用場景 | 流動性穩定的標的 | 大多數標的 |
| 最佳用途 | 快速簡單的執行 | 追蹤市場平均成本 |

### 5.4 高頻交易基礎概念

**高頻交易（HFT）** 利用極低延遲的基礎設施在微秒級別捕獲微小的價格差異。

**主要策略類型：**

| 策略 | 描述 | 持倉時間 |
|------|------|----------|
| **做市（Market Making）** | 同時掛買賣單賺取價差 | 毫秒-秒 |
| **統計套利** | 跨市場、跨品種的微觀價差 | 秒-分鐘 |
| **延遲套利** | 利用不同交易所的報價延遲 | 微秒-毫秒 |
| **事件驅動** | 搶先解讀經濟數據/新聞 | 毫秒-秒 |

- 主機託管（Co-location）：服務器放在交易所旁邊
- FPGA / ASIC：硬件加速訂單處理
- 直連通道（Direct Market Access）：繞過經紀商
- 微波通訊：比光纖更快的跨交易所通訊

### 5.5 做市策略基礎

**核心邏輯：** 做市商同時在買方和賣方掛出限價單，賺取買賣價差（bid-ask spread）。

**基本做市公式：**
```
Bid = Mid - Spread/2 - Skew
Ask = Mid + Spread/2 + Skew
Skew = f(inventory, volatility, order_flow)
```

**庫存風險管理：**
- 做市商持有庫存面臨方向性風險
- 需要根據庫存水平動態調整報價偏移（skew）
- 庫存過多 → 降低 Bid，提高 Ask → 鼓勵市場幫忙減倉

**Avellaneda-Stoikov 模型（2008）：**
```
最優報價 = Mid ± (1/γ) × ln(1 + γ/κ) + γσ²(T-t)/2 × q
```
- γ = 風險厭惡係數
- κ = 訂單到達率
- σ = 波動率
- q = 當前庫存
- T-t = 剩餘時間

### 6.1 回測框架架構

#### 6.1.1 向量化回測（Vectorized Backtesting）

**原理：** 利用 NumPy/Pandas 的矩陣運算一次性計算整個歷史期間的信號與報酬。

**優勢：** 速度極快（比事件驅動快 100-1000 倍）
**劣勢：** 難以模擬真實的交易細節（部分成交、滑點等）

```python
def vectorized_backtest(prices, signals, commission=0.001):
    """
    向量化回測
    signals: +1(做多), -1(做空), 0(空倉)
    """
    returns = prices.pct_change()

    # 策略報酬 = 前一天信號 × 今天報酬
    strategy_returns = signals.shift(1) * returns

    # 扣除交易成本（信號變化時產生）
    trades = signals.diff().abs()
    costs = trades * commission

    net_returns = strategy_returns - costs
    cumulative = (1 + net_returns).cumprod()

    return cumulative, net_returns
```

#### 6.1.2 事件驅動回測（Event-Driven Backtesting）

**原理：** 模擬市場每一根 K 線（或每一筆 Tick），逐步更新持倉、訂單、資金。

**優勢：** 高度真實，可模擬各種訂單類型、部分成交、複雜風控
**劣勢：** 速度慢，開發複雜

```python
class EventDrivenBacktester:
    """
    事件驅動回測引擎（簡化版）
    """
    def __init__(self, initial_capital=1000000):
        self.capital = initial_capital
        self.positions = {}
        self.orders = []
        self.trades = []
        self.equity_curve = []

    def on_bar(self, timestamp, data):
        """每根 K 線觸發"""
        # 1. 更新市場數據
        self._update_market_data(timestamp, data)

        # 2. 處理待執行訂單
        self._process_orders(data)

        # 3. 生成信號
        signal = self.strategy.generate_signal(data)

        # 4. 風控檢查
        if self.risk_manager.check(signal):
            order = self._create_order(signal)
            self.orders.append(order)

        # 5. 記錄權益
        equity = self.capital + self._unrealized_pnl(data)
        self.equity_curve.append({'timestamp': timestamp, 'equity': equity})

    def _process_orders(self, data):
        """模擬訂單執行，含滑點"""
        for order in self.orders:
            fill_price = self._simulate_fill(order, data)
            if fill_price:
                self._execute_trade(order, fill_price)

    def _simulate_fill(self, order, data):
        """模擬成交價格（含滑點）"""
        if order['type'] == 'market':
            slippage = data['atr'] * 0.1  # ATR 的 10% 作為滑點
            if order['side'] == 'buy':
                return data['close'] + slippage
            else:
                return data['close'] - slippage
        return None
```

### 6.2 常見偏差

#### 6.2.1 前視偏差（Look-Ahead Bias）

**定義：** 在回測中使用了當時不可能知道的未來資訊。

**常見來源：**
| 偏差來源 | 範例 | 解決方案 |
|----------|------|----------|
| 使用未來數據計算指標 | 用全樣本均值做標準化 | 只用歷史窗口計算 |
| 財報數據提前使用 | 財報發布前就用該季數據 | 使用 Point-in-Time 數據庫 |
| 復權價格問題 | 用事後復權價計算信號 | 使用前復權或即時復權 |
| 指數成分提前知道 | 用當前成分回溯歷史 | 使用歷史成分股資料 |

#### 6.2.2 生存者偏差（Survivorship Bias）

**定義：** 回測數據只包含存活至今的資產，排除了退市、破產、被收購的標的。

**影響：** 系統性高估報酬，因為失敗的投資在歷史記錄中「消失」了。

**研究顯示：** 含生存者偏差的美股數據年化報酬可能被高估 1-2%。

**解決方案：** 使用 Survivorship-Free 數據庫（如 CRSP、Wind 的歷史全A股數據）。

#### 6.2.3 過擬合偏差（Overfitting Bias）

**定義：** 策略被過度優化以適應歷史數據的噪音，而非真正的市場規律。

**Bailey & Lopez de Prado（2014）** 提出的過擬合公式：

在 N 次回測中找到夏普比率 > 0 的策略之機率：
```
P(最佳策略的夏普 > 0 | 真實夏普 = 0) ≈ 1 - (1 - Φ(0))^N
```
- 嘗試 100 次：99.9% 會找到「看起來有效」的策略
- 嘗試 1000 次：幾乎 100% 會找到

**對策：** 限制回測次數、使用樣本外驗證、提高統計顯著性門檻。

### 6.3 Walk-Forward Analysis

**Walk-Forward（步進式分析）** 是最貼近真實交易的驗證方法。

```
Period 1: |--- Train ---|--- Test ---|
Period 2:      |--- Train ---|--- Test ---|
Period 3:           |--- Train ---|--- Test ---|
...
```

**實現方式：**
1. 訓練窗口（如 3 年）+ 測試窗口（如 6 個月）
2. 在訓練窗口內優化策略參數
3. 在測試窗口內用固定參數測試
4. 滾動向前，重複步驟 1-3
5. 將所有測試窗口的績效串接，作為最終評估

```python
def walk_forward_analysis(data, strategy_fn, train_window=756, test_window=126):
    """
    Walk-Forward 分析
    train_window: 訓練天數（約3年）
    test_window: 測試天數（約6個月）
    """
    results = []
    start = 0

    while start + train_window + test_window <= len(data):
        train_data = data.iloc[start:start + train_window]
        test_data = data.iloc[start + train_window:start + train_window + test_window]

        # 在訓練集上優化參數
        best_params = strategy_fn.optimize(train_data)

        # 在測試集上執行（參數固定）
        test_returns = strategy_fn.execute(test_data, best_params)

        results.append({
            'train_start': train_data.index[0],
            'train_end': train_data.index[-1],
            'test_start': test_data.index[0],
            'test_end': test_data.index[-1],
            'params': best_params,
            'returns': test_returns,
            'sharpe': test_returns.mean() / test_returns.std() * np.sqrt(252)
        })

        start += test_window  # 滾動前進

    return pd.DataFrame(results)
```

### 6.4 Monte Carlo 模擬驗證

**目的：** 評估策略績效是否只是運氣好，以及在不同市場情境下的穩健性。

**方法一：交易序列重排（Bootstrap）**
```python
def monte_carlo_bootstrap(trade_returns, n_simulations=10000):
    """
    透過隨機重排交易序列，模擬不同的績效路徑
    """
    n_trades = len(trade_returns)
    simulations = np.zeros((n_simulations, n_trades))

    for i in range(n_simulations):
        # 隨機抽取（放回）交易報酬
        sampled = np.random.choice(trade_returns, size=n_trades, replace=True)
        simulations[i] = np.cumprod(1 + sampled)

    # 計算信賴區間
    percentiles = np.percentile(simulations, [5, 25, 50, 75, 95], axis=0)

    # 計算破產機率
    final_values = simulations[:, -1]
    ruin_prob = (final_values < 0.5).mean()  # 虧損超過50%的機率

    return percentiles, ruin_prob
```

**方法二：路徑隨機化**
```python
def monte_carlo_path(daily_returns, n_simulations=10000, n_days=252):
    """
    從歷史日報酬中隨機抽取，生成模擬路徑
    """
    simulations = np.zeros((n_simulations, n_days))

    for i in range(n_simulations):
        sampled_days = np.random.choice(daily_returns, size=n_days, replace=True)
        simulations[i] = np.cumprod(1 + sampled_days)

    return simulations
```

### 6.5 交易成本建模

**完整的交易成本模型：**

```python
def total_transaction_cost(
    trade_value,           # 交易金額
    commission_rate=0.001, # 佣金費率（0.1%）
    stamp_tax_rate=0.001,  # 印花稅（僅賣出，A股為0.05%）
    spread_bps=5,          # 買賣價差（基點）
    daily_volume=None,     # 日均成交量（金額）
    daily_volatility=None, # 日波動率
    is_sell=False          # 是否為賣出
):
    """
    計算完整交易成本
    """
    # 1. 佣金
    commission = trade_value * commission_rate

    # 2. 印花稅（部分市場僅賣出徵收）
    stamp_tax = trade_value * stamp_tax_rate if is_sell else 0

    # 3. 買賣價差成本
    spread_cost = trade_value * (spread_bps / 10000) / 2  # 單邊承擔一半

    # 4. 市場衝擊成本（平方根模型）
    impact_cost = 0
    if daily_volume and daily_volatility:
        participation = trade_value / daily_volume
        impact_cost = trade_value * 0.1 * daily_volatility * np.sqrt(participation)

    total = commission + stamp_tax + spread_cost + impact_cost

    return {
        'commission': commission,
        'stamp_tax': stamp_tax,
        'spread_cost': spread_cost,
        'impact_cost': impact_cost,
        'total_cost': total,
        'cost_bps': total / trade_value * 10000
    }
```

### 6.6 績效指標計算

```python
def calculate_performance_metrics(returns, risk_free_rate=0.02):
    """
    計算完整績效指標
    returns: 日報酬率序列
    """
    # 基礎統計
    total_return = (1 + returns).prod() - 1
    annual_return = (1 + returns).prod() ** (252 / len(returns)) - 1
    annual_volatility = returns.std() * np.sqrt(252)

    # 夏普比率
    excess_returns = returns - risk_free_rate / 252
    sharpe_ratio = excess_returns.mean() / returns.std() * np.sqrt(252)

    # 最大回撤
    cumulative = (1 + returns).cumprod()
    running_max = cumulative.cummax()
    drawdown = (cumulative - running_max) / running_max
    max_drawdown = drawdown.min()

    # 最大回撤持續天數
    dd_duration = 0
    max_dd_duration = 0
    for dd in drawdown:
        if dd < 0:
            dd_duration += 1
            max_dd_duration = max(max_dd_duration, dd_duration)
        else:
            dd_duration = 0

    # Calmar Ratio
    calmar_ratio = annual_return / abs(max_drawdown) if max_drawdown != 0 else np.inf

    # Sortino Ratio（只計算下行波動）
    downside_returns = returns[returns < 0]
    downside_std = downside_returns.std() * np.sqrt(252)
    sortino_ratio = (annual_return - risk_free_rate) / downside_std if downside_std > 0 else np.inf

    # 勝率
    win_rate = (returns > 0).mean()

    # 盈虧比
    avg_win = returns[returns > 0].mean() if (returns > 0).any() else 0
    avg_loss = abs(returns[returns < 0].mean()) if (returns < 0).any() else 1
    profit_loss_ratio = avg_win / avg_loss if avg_loss > 0 else np.inf

    # 盈利因子
    gross_profit = returns[returns > 0].sum()
    gross_loss = abs(returns[returns < 0].sum())
    profit_factor = gross_profit / gross_loss if gross_loss > 0 else np.inf

    # VaR (95%)
    var_95 = np.percentile(returns, 5)

    # CVaR / Expected Shortfall (95%)
    cvar_95 = returns[returns <= var_95].mean()

    return {
        '總報酬率': f'{total_return:.2%}',
        '年化報酬率': f'{annual_return:.2%}',
        '年化波動率': f'{annual_volatility:.2%}',
        '夏普比率': f'{sharpe_ratio:.2f}',
        '最大回撤': f'{max_drawdown:.2%}',
        '最大回撤天數': max_dd_duration,
        'Calmar Ratio': f'{calmar_ratio:.2f}',
        'Sortino Ratio': f'{sortino_ratio:.2f}',
        '勝率': f'{win_rate:.2%}',
        '盈虧比': f'{profit_loss_ratio:.2f}',
        '盈利因子': f'{profit_factor:.2f}',
        'VaR (95%)': f'{var_95:.2%}',
        'CVaR (95%)': f'{cvar_95:.2%}'
    }
```

## 七、風控與資金管理

### 7.1 凱利公式（Kelly Criterion）

**核心邏輯：** 由 John Kelly（1956）提出，計算使長期幾何成長率最大化的最優下注比例。

**基本公式：**
```
f* = (p × b - q) / b = W - (1 - W) / R
```
- f* = 最優資金比例
- p (W) = 勝率
- q = 1 - p（敗率）
- b (R) = 賠率（平均獲利 / 平均虧損）

**範例：**
```
勝率 = 55%, 賠率 = 1.5
f* = (0.55 × 1.5 - 0.45) / 1.5 = 0.825 / 1.5 = 25%
```

**連續情境的凱利公式（正態分布假設）：**
```
f* = μ / σ²
```
- μ = 期望報酬率
- σ² = 報酬率方差

**多資產凱利（投資組合版本）：**
```
f* = Σ⁻¹ × μ
```
- Σ⁻¹ = 共變異數矩陣的逆矩陣
- μ = 期望報酬向量

**關鍵警告：全凱利（Full Kelly）的回撤風險極大！**

研究顯示：
- 全凱利（100%）可能產生 -61.5% 的最大回撤
- 半凱利（50%）回撤降至約 -24.6%，報酬僅降低一半
- 四分之一凱利（25%）情緒上最可承受

**風險約束凱利（Risk-Constrained Kelly）：**

Busseti 等（2016）提出在凱利公式中加入回撤約束：
```
max f: E[log(1 + f × R)]
s.t. P(Drawdown > D_max) ≤ α
```

```python
def kelly_fraction(win_rate, avg_win, avg_loss, fraction=0.5):
    """
    計算凱利比例（支持分數凱利）
    fraction: 使用全凱利的比例（0.5 = 半凱利）
    """
    if avg_loss == 0:
        return 0

    b = avg_win / avg_loss  # 賠率
    q = 1 - win_rate

    full_kelly = (win_rate * b - q) / b

    # 不允許負值（不值得下注）
    full_kelly = max(0, full_kelly)

    return full_kelly * fraction

def portfolio_kelly(expected_returns, cov_matrix, fraction=0.25):
    """
    多資產凱利公式
    """
    cov_inv = np.linalg.inv(cov_matrix)
    full_kelly_weights = cov_inv @ expected_returns

    return full_kelly_weights * fraction
```

### 7.2 固定比例風險法

**核心邏輯：** 每筆交易承擔固定比例的帳戶資金風險（通常 0.5%-2%）。

```
倉位大小 = (帳戶資金 × 風險比例) / (進場價 - 止損價)
```

**範例：**
```
帳戶 = 1,000,000
風險比例 = 1%
進場價 = 100
止損價 = 95
倉位 = (1,000,000 × 1%) / (100 - 95) = 10,000 / 5 = 2,000 股
```

```python
def fixed_fractional_position_size(
    account_value,
    risk_pct,
    entry_price,
    stop_price
):
    """
    固定比例風險法計算倉位
    """
    risk_amount = account_value * risk_pct
    risk_per_share = abs(entry_price - stop_price)

    if risk_per_share == 0:
        return 0

    shares = int(risk_amount / risk_per_share)
    position_value = shares * entry_price

    return {
        'shares': shares,
        'position_value': position_value,
        'risk_amount': risk_amount,
        'position_pct': position_value / account_value
    }
```

**優勢：**
- 簡單易行，適合所有策略類型
- 自動縮放（賺錢時自動加大，虧錢時自動減小）
- 有效控制單筆最大損失

**劣勢：**
- 不考慮各交易間的相關性
- 止損距離影響倉位大小，窄止損可能導致過大倉位

### 7.3 最大回撤控制

**最大回撤（Maximum Drawdown, MDD）** 是衡量策略風險最重要的指標之一。

**動態回撤控制機制：**

```python
class DrawdownController:
    """
    動態回撤控制器
    """
    def __init__(self, max_drawdown_limit=0.15, warning_level=0.10):
        self.max_drawdown_limit = max_drawdown_limit  # 最大允許回撤
        self.warning_level = warning_level             # 預警水位
        self.peak_equity = 0
        self.current_equity = 0

    def update(self, equity):
        """更新權益並返回風險狀態"""
        self.current_equity = equity
        self.peak_equity = max(self.peak_equity, equity)

        current_dd = (self.peak_equity - self.current_equity) / self.peak_equity

        if current_dd >= self.max_drawdown_limit:
            return 'STOP', 0.0  # 全面停止交易
        elif current_dd >= self.warning_level:
            # 線性減倉：回撤越大，倉位越小
            scale = 1 - (current_dd - self.warning_level) / (self.max_drawdown_limit - self.warning_level)
            return 'REDUCE', max(0.1, scale)  # 最少保留10%倉位
        else:
            return 'NORMAL', 1.0  # 正常交易

    def get_position_scale(self, equity):
        """獲取當前應使用的倉位縮放比例"""
        status, scale = self.update(equity)
        return scale
```

### 7.4 止損策略設計

#### 7.4.1 固定比例止損

```
止損價 = 進場價 × (1 - 止損比例)
```
- 常用比例：3%-8%
- 優勢：簡單直觀
- 劣勢：不考慮波動率，可能在高波動時被頻繁觸發

#### 7.4.2 ATR 止損

```
止損價 = 進場價 - N × ATR
```
- N 通常取 1.5-3.0
- 優勢：自動適應市場波動
- 劣勢：波動劇增時止損距離擴大

#### 7.4.3 時間止損

```
若持倉 T 天後仍未達到獲利目標，則無條件平倉
```
- 常用時間：10-30 個交易日
- 適用場景：均值回歸策略、事件驅動策略
- 優勢：避免資金長期被鎖定在無效交易中

#### 7.4.4 追蹤止損（Trailing Stop）

```
止損價 = max(歷史最高價) - N × ATR
```
隨著價格上漲，止損價也跟著上移，但永遠不會下移。

```python
def trailing_stop(prices, atr_values, n_atr=2.5):
    """
    ATR 追蹤止損
    """
    stop_levels = pd.Series(index=prices.index, dtype=float)
    highest = prices.iloc[0]

    for i in range(len(prices)):
        highest = max(highest, prices.iloc[i])
        stop_levels.iloc[i] = highest - n_atr * atr_values.iloc[i]

        # 如果觸發止損，重置
        if prices.iloc[i] < stop_levels.iloc[i]:
            highest = prices.iloc[i]  # 重置

    return stop_levels
```

**止損策略比較：**

| 方法 | 適用策略 | 優勢 | 劣勢 |
|------|----------|------|------|
| 固定比例 | 所有策略 | 簡單、可控 | 不適應波動率 |
| ATR 止損 | 趨勢跟蹤 | 適應波動率 | 波動劇增時距離太遠 |
| 時間止損 | 均值回歸 | 避免資金鎖定 | 可能錯過延遲回歸 |
| 追蹤止損 | 趨勢跟蹤 | 保護浮盈 | 趨勢中途被震出 |

### 7.5 投資組合層面風控

#### 7.5.1 風險預算法（Risk Budgeting）

```python
def risk_parity_weights(cov_matrix):
    """
    風險平價（Risk Parity）：使每個資產對組合風險的貢獻相等
    """
    from scipy.optimize import minimize

    n = cov_matrix.shape[0]

    def risk_contribution(weights):
        portfolio_vol = np.sqrt(weights @ cov_matrix @ weights)
        marginal_risk = cov_matrix @ weights
        risk_contrib = weights * marginal_risk / portfolio_vol
        return risk_contrib

    def objective(weights):
        rc = risk_contribution(weights)
        target_rc = np.ones(n) / n  # 等風險貢獻
        return np.sum((rc - target_rc) ** 2)

    constraints = [{'type': 'eq', 'fun': lambda w: np.sum(w) - 1}]
    bounds = [(0.01, 0.5) for _ in range(n)]
    x0 = np.ones(n) / n

    result = minimize(objective, x0, method='SLSQP',
                     bounds=bounds, constraints=constraints)

    return result.x
```

```
第一層：單筆交易風控

第二層：策略層面風控
```

### 7.6 相關性監控與動態調整

**核心問題：** 在正常市場中低相關的資產，在危機時可能同時下跌（相關性趨近 1）。

```python
def dynamic_correlation_monitor(returns, lookback=60, threshold=0.7):
    """
    動態相關性監控
    當相關性異常升高時發出警報
    """
    rolling_corr = returns.rolling(lookback).corr()

    warnings = []
    for date in returns.index[lookback:]:
        corr_matrix = rolling_corr.loc[date]
        # 取上三角元素（排除對角線）
        upper_tri = corr_matrix.values[np.triu_indices_from(corr_matrix, k=1)]
        avg_corr = upper_tri.mean()
        max_corr = upper_tri.max()

        if avg_corr > threshold:
            warnings.append({
                'date': date,
                'avg_correlation': avg_corr,
                'max_correlation': max_corr,
                'action': '建議降低總倉位至50%以下'
            })

    return pd.DataFrame(warnings)

def adaptive_position_sizing(base_weights, correlation_level):
    """
    根據相關性水平動態調整倉位
    """
    if correlation_level > 0.8:
        scale = 0.3  # 高相關性 → 大幅降倉
    elif correlation_level > 0.6:
        scale = 0.6  # 中等相關性 → 適度降倉
    elif correlation_level > 0.4:
        scale = 0.8  # 正常水平 → 略微降倉
    else:
        scale = 1.0  # 低相關性 → 正常倉位

    adjusted_weights = base_weights * scale
    return adjusted_weights
```
