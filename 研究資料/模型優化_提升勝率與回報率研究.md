### 1.2 已知問題深度剖析

#### 問題一：TSLA Q4看跌卻暴漲80%（事件驅動盲區）

**根本原因分析：**
- 模型完全依賴「歷史統計量」（估值、動量、技術面），無法捕捉非連續性事件
- TSLA在2024 Q4的暴漲主要來自：政策預期（馬斯克與政治關聯）、AI機器人概念重估、市場情緒急劇轉向
- 傳統估值框架（PE>60視為過熱）對TSLA類公司系統性失效

**解決方向：**
- 加入新聞/社交媒體情緒信號，捕捉「敘事變化」
- 加入期權市場信號（異常看漲期權活動可提前偵測）
- 對「概念股/敘事驅動型」個股建立獨立評估框架

#### 問題二：JNJ兩次誤判看漲

**根本原因分析：**
- JNJ基本面數據（ROE、FCF、股息率）持續優良，質量分數和價值分數高
- 但模型忽略了：訴訟風險（talc案件）、業務分拆影響、市場風格切換（防禦轉攻擊）
- 分析師評級可能滯後（分析師傾向維持既有評級）

**解決方向：**
- 加入法律/監管風險信號（新聞情緒可捕捉）
- 對分析師評級做「變化率」分析而非絕對值
- 考慮市場風格偏好（risk-on環境中防禦股通常表現不佳）

#### 問題三：ADBE持續誤判

**根本原因分析：**
- ADBE估值長期偏高（PS>10），但模型對科技股的容忍度設定可能不夠精細
- AI競爭衝擊（Canva、免費AI設計工具）改變了成長預期
- 分析師目標價可能過度樂觀

**解決方向：**
- 引入「競爭威脅」評估（可通過替代數據如App下載量、網站流量監控）
- 加入盈餘修正（Earnings Revision）因子 — 分析師下調盈餘預期是強看空信號
- 對DCF估值增加「情境分析」（base/bull/bear case）

#### 問題四：期間3（2024 Q4）整體表現較差

**根本原因分析：**
- 2024 Q4市場環境特殊：美國大選後Risk-On急劇升溫、VIX快速下降
- 模型的宏觀環境判斷可能滯後（用的是當前VIX而非預期）
- 低質量股在risk-on環境中反而大漲（「垃圾股反彈」現象）

**解決方向：**
- 加入VIX期限結構（contango vs backwardation）作為前瞻性指標
- 在不同市場環境中使用完全不同的策略（而非僅調整權重）
- 加入「市場寬度」指標（advance-decline ratio、% above 200MA）

#### 問題五：平均回報從v1的11.45%降到v2的9.56%

**根本原因分析：**
- v2增加更多數據源後，模型變得更「保守」（更多信號 = 更多衝突 = 傾向中性）
- 技術面否決機制可能過度保守（tech_score < -30 即減15%）
- 更多預測被發出（v2有81筆 vs v1可能更少），包含更多邊際案例

**解決方向：**
- 調整信號合成邏輯：從「等權加總」改為「信心加權」
- 只在高信心時出手（提高閾值，減少邊際交易）
- 最佳化出場策略（目前回測用固定時間窗口，缺乏動態出場）

## 二、信號增強方法

### 2.1 新聞情緒/NLP信號

#### 2.1.1 最新研究成果

**FinBERT vs LLM比較（2024-2025研究）：**

| 模型 | 情緒分類準確率 | 年化BHR | 特點 |
|------|--------------|---------|------|
| VADER（傳統） | ~65% | ~5% | 速度快，但理解淺 |
| FinBERT | 72.2% | 5.98% | 金融領域專精 |
| OPT (GPT-3) | 74.4% | ~7% | 通用LLM |
| LLaMA-2（微調） | ~78% | 12.37% | 最新最強 |
| FinBERT-LSTM混合 | ~80% | — | 結合時序預測 |

**關鍵發現：**
- LLaMA-2在金融情緒分析中顯著優於FinBERT，年化報酬12.37% vs 5.98%（來源：ScienceDirect, 2024）
- FinBERT在動蕩市場中的預測力下降，而LLaMA-2模型保持穩健（來源：同上）
- FinBERT-LSTM結合模型在股價預測中表現最佳（來源：ACM, 2024）
- GPT-4結合FinBERT和邏輯回歸的混合方法達到最高準確率（來源：MDPI Big Data and Cognitive Computing, 2024）
- 新聞情緒因子的IC（Information Coefficient）通常在0.02-0.05之間，雖然看似微小，但在大樣本中具統計顯著性

**財報電話會議語氣分析：**
- 管理層語氣變化（當季 vs 過去四季平均）是有效的預測信號
- UC Berkeley 2024研究分析1,900+份NASDAQ前50大公司的電話會議紀錄，確認語義分析可預測次日股價走勢
- 「語氣突變」（tone surprise）比絕對語氣更有預測力

#### 2.1.2 社交媒體情緒

**Reddit/WallStreetBets：**
- Reddit討論對突發波動的預測力 > Twitter（Twitter更適合漸進式市場反應）
- **重要發現：社交媒體「情緒」的相關性其實很弱；但「討論量」和Google搜索趨勢的預測信號更強**
- WSB高關注度時建立的部位，持有期回報為8.5%，但整體上WSB關注度增加會降低持有期回報
- 結論：社交媒體數據應以「關注度/交易量異常」為主要信號，而非情緒正負面

**實務建議：**
- 監控「情緒突變」（sentiment shock）而非絕對情緒值
- 結合討論量 + 情緒方向 + Google Trends的三維信號
- 對大盤/權值股效果較好，對小盤股噪音太多

#### 2.1.3 事件驅動信號構建

**PEAD（財報公布後漂移）：**
- 做多SUE（標準化意外盈餘）前10%、做空後10%的對沖組合，三個月風險調整回報5.1%，年化超過20%
- 但PEAD在美股中的效應近年在衰減（市場效率提升、機構投資者搶先交易）
- Garfinkel, Hribar & Hsiao (2024) 使用CNN分析盈餘數據的視覺化圖表來預測PEAD，提供了正交於傳統預測因子的新信號

**建議的事件驅動信號體系：**

| 事件類型 | 信號構建方式 | 預期IC | 實現難度 |
|---------|------------|--------|---------|
| 盈餘驚喜 | SUE + 盈餘修正方向 | 0.03-0.06 | 低 |
| 管理層語氣 | LLM分析電話會議紀錄 | 0.02-0.04 | 中 |
| 分析師修正 | 盈餘/目標價調升降 | 0.04-0.07 | 低 |
| 內部人交易 | 已有，可強化 | 0.02-0.03 | 已實現 |
| 監管/法律事件 | 新聞NLP篩選 | 0.01-0.03 | 中 |
| 併購/分拆 | 公告監控 | 0.05-0.10 | 中 |

#### 2.1.4 預期效果

加入新聞情緒信號後的預期提升：
- **勝率提升：+3-5%**（從67.9%到71-73%）
- **回報提升：+1-2%**
- 主要貢獻在於避免TSLA類的重大誤判（事件驅動捕捉）和JNJ類的風險忽略

### 2.2 另類數據信號

#### 2.2.1 市場規模與採用率

- 全球另類數據市場2025年規模約140-180億美元，CAGR超過50%
- 2024年67%的投資經理已將另類數據納入策略，94%的用戶計劃增加預算
- J.P. Morgan 2024研究：使用另類數據的對沖基金年化回報高出3%

#### 2.2.2 各類另類數據的Alpha

| 數據類型 | 預測改善幅度 | 數據可得性 | 成本 | 推薦優先級 |
|---------|------------|-----------|------|-----------|
| 社交媒體情緒 | 預測準確率87% | 高（API可得） | 低-中 | 高 |
| 信用卡消費數據 | 預測改善10% | 中（需購買） | 高 | 中 |
| 衛星圖像 | 盈餘預測改善18% | 低（專業供應商） | 很高 | 低 |
| 網絡流量/App下載 | 可作為營收先行指標 | 高（SimilarWeb等） | 中 | 高 |
| 招聘數據 | 成長前瞻指標 | 高（LinkedIn） | 中 | 中 |
| 專利申請 | 創新能力評估 | 高（公開數據） | 低 | 低 |

#### 2.2.3 實務建議

**對個人投資者而言最具性價比的另類數據：**

1. **網絡流量數據**（SimilarWeb免費版可用）：監控ADBE、CRM等SaaS公司的網站流量變化
2. **App下載數據**（Sensor Tower / data.ai）：監控消費類、科技類公司
3. **招聘職缺數據**（Indeed、LinkedIn）：職缺增減是營收變化的1-2季先行指標
4. **Google Trends**：免費且即時，關注度突變是重要信號

### 2.3 期權市場信號

#### 2.3.1 Put/Call Ratio

- 正向（負向）的Call-Put IV價差與隨後4週的顯著正（負）異常回報相關
- 預測力源自期權市場中知情交易者的存在
- 但隨著投資者學會更快將期權信息納入股價，預測力在逐步下降

#### 2.3.2 Implied Volatility Skew

- 向下傾斜的偏斜暗示潛在的下行價格走勢；向上的偏斜暗示上行偏向和價格上漲預期
- Skew的嚴重程度是市場情緒的函數，在動蕩或看跌市場中特別顯著
- **實務應用**：將IV Skew作為方向性過濾器 — 當skew極端看空時，即使基本面好也應謹慎

#### 2.3.3 Gamma Exposure (GEX)

- **正GEX**：做市商對沖時產生穩定效應（壓縮波動）
- **負GEX**：做市商對沖時放大波動（助漲助跌）
- GEX可以標識重要的支撐/阻力位（做市商集中對沖的價位）
- **實務案例**：2025年11月17日，SPY在670 strike的GEX暴露快速累積，之後SPY精確回測該GEX水平並急跌

#### 2.3.4 異常期權活動（Unusual Options Activity）

- 大額看漲期權買入（特別是短期虛值看漲）通常先於重大上漲
- 適用於捕捉TSLA類事件驅動行情
- 需要期權數據訂閱（如CBOE、OptionMetrics）

**期權信號的預期效果：**
- **勝率提升：+2-3%**
- **主要價值在於「避雷」和「捕捉異常」**，而非日常預測

## 三、模型架構優化

### 3.1 機器學習替代線性評分

#### 3.1.1 XGBoost/LightGBM實證表現

**最新研究數據（2024-2025）：**

| 模型 | 累積回報 | 年化回報 | 特點 |
|------|---------|---------|------|
| XGBoost | 11.31%（6個月） | 20.81% | 特徵重要性可解釋 |
| LightGBM | 10.74%（6個月） | 19.73% | 訓練速度更快 |
| XGBoost+LightGBM融合 | 優於單一模型 | — | 1:1融合效果最佳 |
| 傳統OLS因子模型 | 低於ML方法 | — | 基準比較 |

**關鍵發現：**
- LightGBM在中國A股市場的長空策略顯著優於傳統OLS方法和市場基準
- XGBoost+LightGBM 1:1融合預測效果優於任何單一模型
- ML方法在S&P 500上的表現比傳統因子模型高出年化3-5%

#### 3.1.2 避免過擬合的最佳實踐

**Walk-Forward驗證（最推薦）：**
```
訓練期: [T-24月, T-6月]  →  測試期: [T-6月, T]
然後滾動：
訓練期: [T-21月, T-3月]  →  測試期: [T-3月, T+3月]
...以此類推
```

**Purged K-Fold交叉驗證：**
- 在訓練集和測試集之間加入「隔離帶」（purge gap），避免信息洩漏
- 通常purge gap = 預測持有期的2倍

**特徵選擇最佳方法：**
1. SHAP值排序（最推薦） — 可解釋性 + 穩定性
2. 遞歸特徵消除（RFE）
3. L1正則化（Lasso）自動篩選
4. 前向/後向逐步選擇

#### 3.1.3 集成學習方法

**推薦方案：Stacking集成**
```
第一層：
  - LightGBM（捕捉非線性交互）
  - XGBoost（特徵重要性穩健）
  - 線性Ridge回歸（保持穩定性）
第二層（元學習器）：
  - 簡單線性模型（避免過擬合）
```

**預期效果：**
- 比單一模型的勝率再提升2-4%
- 回報穩定性顯著提升（降低最大回撤）

### 3.2 時間序列深度學習

#### 3.2.1 LSTM

- LSTM在短期預測中表現尚可，但長期預測中不如Transformer
- LSTM-DQL混合模型年化回報45.12%、夏普比率1.51（但需注意可能存在過擬合）
- **實務建議**：LSTM適合作為技術面信號的補充，但不建議作為核心模型

#### 3.2.2 Temporal Fusion Transformer (TFT)

**最新實證（2024-2025）：**
- TFT模型在2024年1月至2025年9月的回測中，從$1增長至$9.07（+807%），遠超AAPL的+38%（買入持有）
- TFT結合自適應夏普比率優化，在所有市場環境中保持優越表現，高波動期尤其突出
- TFT-GNN（圖神經網路）混合模型展示了最前沿的研究方向

**Transformer vs LSTM比較：**
- LSTM在短期預測中表現較好
- Vanilla Transformer在長期預測中表現更好
- CNN-LSTM-TFT混合架構在準確性和可解釋性上最優

**實務建議：**
- TFT的工程複雜度高，建議作為中長期研究方向
- 短期可先用LightGBM/XGBoost替代線性評分，已能獲得大部分改善

## 四、風控與出場策略優化

### 4.1 動態止損

#### 4.1.1 各類止損方法比較

| 止損方法 | 適用場景 | 優點 | 缺點 | 推薦ATR倍數 |
|---------|---------|------|------|------------|
| 固定百分比止損 | 簡單策略 | 易實現 | 不適應波動 | — |
| ATR追蹤止損 | 趨勢跟蹤 | 自適應波動 | 盤整期頻繁觸發 | 2.5-3x |
| Chandelier Exit | 趨勢市 | 從高點回撤 | 需要趨勢確認 | 3x |
| Parabolic SAR | 短期交易 | 自動加速 | 震盪市假信號多 | — |
| 時間止損 | 所有策略 | 避免資金沉澱 | 可能過早退出 | — |

#### 4.1.2 最優止損比例研究

**基於持有期的最優止損：**
- **1個月持有期**：ATR 2.0x（約8-10%固定止損）
- **3個月持有期**：ATR 2.5x（約12-15%固定止損）
- **6個月持有期**：ATR 3.0x（約15-20%固定止損）

**當前模型的止損問題：**
- 固定8%止損太窄 — 高波動股（如TSLA）經常觸發後反彈
- 建議改為：ATR自適應止損 + 時間止損的組合

#### 4.1.3 建議方案

```
動態止損 = max(
    ATR(14) * 2.5,              # 波動自適應
    固定最低止損 6%,              # 底線保護
    min(固定最高止損 20%, ...)    # 上限控制
)

時間止損：
- 持倉超過預定持有期150%仍無明顯盈利 → 強制出場
```

### 4.2 獲利了結策略

#### 4.2.1 研究發現

**關鍵洞察：**
- 複雜的出場策略（嚴格止損、追蹤止損、利潤目標）在回測中並不穩定地優於簡單出場
- 「保持簡單」是最重要的原則 — 變量越多，過擬合風險越大
- 分批出場策略是兼顧穩定性和收益的最佳實踐

**推薦的分批獲利方案：**

```
第一批（50%）：達到目標回報的60%時出場
  例：目標10% → 6%時賣一半
第二批（30%）：達到目標回報的100%時出場
第三批（20%）：使用追蹤止損（ATR 2x）讓利潤奔跑
```

**預期效果：**
- 提高實現回報率約1.5-2%（減少「坐電梯」情況）
- 降低最大回撤約2-3%

#### 4.2.2 Triple Barrier方法

**推薦採用Marcos Lopez de Prado的Triple Barrier標籤法：**
- **上界**：利潤目標（如+15%）
- **下界**：止損線（如-10%或ATR 2.5x）
- **時間界**：最大持有期（如90天）
- 先觸碰哪個界限就以哪個為出場信號

### 4.3 倉位管理優化

#### 4.3.1 凱利公式的實務應用

**公式：** f* = (bp - q) / b
- 其中 b = 平均贏虧比, p = 勝率, q = 1-p

**當前模型的凱利值：**
- 勝率 p = 67.9%, q = 32.1%
- 假設平均贏虧比 b = 1.5（需從回測數據精確計算）
- f* = (1.5 * 0.679 - 0.321) / 1.5 = (1.0185 - 0.321) / 1.5 = 46.5%

**實務建議：**
- 全凱利（46.5%）風險過高，波動劇烈
- **半凱利（23%）**：年化回報接近最大化，但波動仍高
- **四分之一凱利（12%）**：更穩定的增長，推薦用於實盤
- 最近的回測研究顯示，受約束的凱利策略回報2.53%-17.24%，風控表現穩健

#### 4.3.2 信心加權倉位

**推薦方案：將模型信心度與倉位大小掛鉤**

| 總分範圍 | 信心等級 | 倉位比例（佔可投資資金） |
|---------|---------|---------------------|
| >= 78 | 高信心 | 12-15%（四分之一凱利上限） |
| 68-77 | 中高信心 | 8-10% |
| 60-67 | 中信心 | 5-7% |
| 55-59 | 低信心 | 3-4% |
| < 55 | 不交易 | 0% |

#### 4.3.3 集中 vs 分散持倉

**研究結論：**
- 大盤股組合中，10支股票的平均波動率20%，40支股票僅降至17%（增加30支只降低3%）
- 大盤股組合超過15支後，分散化邊際效益極低
- 基金經理集中持倉的前幾大部位，年化跑贏分散型基金約4%
- **結論：10-15支高信心集中持倉 > 30支分散持倉**

**推薦方案：**
- 核心持倉：8-12支高信心股票（每支8-15%）
- 衛星持倉：3-5支中信心股票（每支3-5%）
- 總持倉：不超過15支

#### 5.1.1 等待回撤 vs 立即建倉

## 八、Python實現方案

### 8.1 盈餘修正因子（P0 — 最高優先級）

```python
import requests
from datetime import datetime, timedelta

class EarningsRevisionFactor:
    """盈餘修正因子：追蹤分析師盈餘預測的修正方向"""

    def __init__(self, fmp_key):
        self.fmp_key = fmp_key
        self.base_url = "https://financialmodelingprep.com/stable"

    def get_earnings_revision_score(self, ticker):
        """
        計算盈餘修正分數 (0-100)
        核心邏輯：過去90天內分析師盈餘預測的修正方向
        """
        score = 50  # 預設中性
        reasons = []

        try:
            # 獲取分析師預測數據
            url = f"{self.base_url}/analyst-estimates?symbol={ticker}&apikey={self.fmp_key}"
            r = requests.get(url, timeout=10)
            data = r.json()

            if not data or len(data) < 2:
                return score, reasons

            # 比較最近兩期的預測變化
            current = data[0]  # 最近期
            previous = data[1]  # 前一期

            # EPS預測變化
            current_eps = current.get('estimatedEpsAvg', 0)
            previous_eps = previous.get('estimatedEpsAvg', 0)

            if previous_eps and previous_eps != 0:
                eps_revision = (current_eps - previous_eps) / abs(previous_eps)

                if eps_revision > 0.10:
                    score = 90
                    reasons.append(f"EPS預測大幅上調({eps_revision*100:.1f}%)")
                elif eps_revision > 0.03:
                    score = 75
                    reasons.append(f"EPS預測上調({eps_revision*100:.1f}%)")
                elif eps_revision > 0:
                    score = 60
                elif eps_revision < -0.10:
                    score = 15
                    reasons.append(f"EPS預測大幅下調({eps_revision*100:.1f}%)")
                elif eps_revision < -0.03:
                    score = 30
                    reasons.append(f"EPS預測下調({eps_revision*100:.1f}%)")
                elif eps_revision < 0:
                    score = 40

            # 營收預測變化
            current_rev = current.get('estimatedRevenueAvg', 0)
            previous_rev = previous.get('estimatedRevenueAvg', 0)

            if previous_rev and previous_rev != 0:
                rev_revision = (current_rev - previous_rev) / abs(previous_rev)

                if rev_revision > 0.05:
                    score = min(score + 10, 100)
                    reasons.append(f"營收預測上調({rev_revision*100:.1f}%)")
                elif rev_revision < -0.05:
                    score = max(score - 10, 0)
                    reasons.append(f"營收預測下調({rev_revision*100:.1f}%)")

            # 分析師數量變化（更多分析師關注 = 信號更可靠）
            current_count = current.get('numberAnalystEstimatedEps', 0)
            if current_count and current_count > 10:
                # 高覆蓋度，信號更可靠
                if score > 60:
                    score = min(score + 5, 100)
                elif score < 40:
                    score = max(score - 5, 0)

        except Exception as e:
            pass
```

### 8.2 Triple Barrier出場策略（P0）

```python
import pandas as pd
import numpy as np

class TripleBarrierExit:
    """Triple Barrier出場策略：上界(利潤) + 下界(止損) + 時間界"""

    def __init__(self):
        pass

    def calculate_barriers(self, entry_price, atr, confidence_level="medium"):
        """
        根據ATR和信心等級計算三重障礙

        參數：
            entry_price: 進場價格
            atr: 14日ATR值
            confidence_level: "high" / "medium" / "low"

        回傳：
            dict: 包含上界、下界、時間界
        """
        # 基於信心的ATR倍數
        multipliers = {
            "high":   {"take_profit": 4.0, "stop_loss": 2.5, "max_days": 120},
            "medium": {"take_profit": 3.0, "stop_loss": 2.0, "max_days": 90},
            "low":    {"take_profit": 2.0, "stop_loss": 1.5, "max_days": 60},
        }

        m = multipliers.get(confidence_level, multipliers["medium"])

        # 止損不低於6%、不高於20%
        raw_stop = atr * m["stop_loss"]
        stop_pct = max(0.06, min(0.20, raw_stop / entry_price))

        # 利潤目標
        raw_tp = atr * m["take_profit"]
        tp_pct = max(0.08, raw_tp / entry_price)

        return {
            "take_profit_price": entry_price * (1 + tp_pct),
            "take_profit_pct": round(tp_pct * 100, 1),
            "stop_loss_price": entry_price * (1 - stop_pct),
            "stop_loss_pct": round(stop_pct * 100, 1),
            "max_holding_days": m["max_days"],
        }

    def simulate_exit(self, prices_series, entry_price, barriers):
        """
        模擬Triple Barrier出場

        參數：
            prices_series: 進場後的每日價格序列
            entry_price: 進場價格
            barriers: calculate_barriers的回傳值

        回傳：
            dict: 出場結果
        """
        tp = barriers["take_profit_price"]
        sl = barriers["stop_loss_price"]
        max_days = barriers["max_holding_days"]

        # 分批出場追蹤
        remaining_position = 1.0  # 100%持倉
        total_realized = 0.0
        exit_log = []

        # 第一批出場點：利潤目標的60%
        first_target = entry_price * (1 + barriers["take_profit_pct"] / 100 * 0.6)
        first_batch_sold = False

        # 第二批出場點：利潤目標的100%
        second_batch_sold = False

        for i, price in enumerate(prices_series):
            day = i + 1

            if remaining_position <= 0:
                break

            # 止損檢查（對所有剩餘持倉）
            if price <= sl:
                realized = (price / entry_price - 1) * remaining_position
                total_realized += realized
                exit_log.append({
                    "day": day, "type": "止損",
                    "price": price, "portion": remaining_position,
                    "return_pct": round((price / entry_price - 1) * 100, 2)
                })
                remaining_position = 0
                break

            # 第一批：達到60%目標時賣出50%
            if not first_batch_sold and price >= first_target:
                portion = 0.50
                realized = (price / entry_price - 1) * portion
                total_realized += realized
                remaining_position -= portion
                first_batch_sold = True
                exit_log.append({
                    "day": day, "type": "第一批獲利",
                    "price": price, "portion": portion,
                    "return_pct": round((price / entry_price - 1) * 100, 2)
                })
                # 止損上移至成本價（保本止損）
                sl = entry_price

            # 第二批：達到100%目標時賣出30%
            if first_batch_sold and not second_batch_sold and price >= tp:
                portion = 0.30
                realized = (price / entry_price - 1) * portion
                total_realized += realized
                remaining_position -= portion
                second_batch_sold = True
                exit_log.append({
                    "day": day, "type": "第二批獲利",
                    "price": price, "portion": portion,
                    "return_pct": round((price / entry_price - 1) * 100, 2)
                })
                # 止損再上移至第一批出場價
                sl = first_target

            # 時間止損
            if day >= max_days and remaining_position > 0:
                realized = (price / entry_price - 1) * remaining_position
                total_realized += realized
                exit_log.append({
                    "day": day, "type": "時間止損",
                    "price": price, "portion": remaining_position,
                    "return_pct": round((price / entry_price - 1) * 100, 2)
                })
                remaining_position = 0
                break

        # 如果還有剩餘持倉（追蹤止損部分）
        if remaining_position > 0 and len(prices_series) > 0:
            final_price = prices_series.iloc[-1] if hasattr(prices_series, 'iloc') else prices_series[-1]
            realized = (final_price / entry_price - 1) * remaining_position
            total_realized += realized
            exit_log.append({
                "day": len(prices_series), "type": "期末結算",
                "price": final_price, "portion": remaining_position,
                "return_pct": round((final_price / entry_price - 1) * 100, 2)
            })

        return {
            "total_return_pct": round(total_realized * 100, 2),
            "exit_log": exit_log,
            "hit_take_profit": second_batch_sold,
            "hit_stop_loss": any(e["type"] == "止損" for e in exit_log),
            "hit_time_stop": any(e["type"] == "時間止損" for e in exit_log),
        }
```

### 8.3 LightGBM集成模型（P1）

```python
import lightgbm as lgb
import xgboost as xgb
import numpy as np
import pandas as pd
from sklearn.linear_model import Ridge
from sklearn.model_selection import TimeSeriesSplit
from sklearn.metrics import accuracy_score, classification_report

class MLEnsemblePredictor:
    """
    ML集成預測模型：LightGBM + XGBoost + Ridge Stacking
    替代現有的線性加權評分
    """

    def __init__(self):
        self.lgb_model = None
        self.xgb_model = None
        self.ridge_model = None
        self.meta_model = None
        self.feature_names = None
        self.is_trained = False

    def prepare_features(self, analysis_result):
        """
        將現有v2分析結果轉換為ML特徵向量
        """
        features = {
            # 現有因子分數
            'quality_score': analysis_result.get('quality_score', 50),
            'value_score': analysis_result.get('value_score', 50),
            'momentum_score': analysis_result.get('momentum_score', 50),
            'analyst_score': analysis_result.get('analyst_score', 50),
            'insider_score': analysis_result.get('insider_score', 50),
            'dcf_score': analysis_result.get('dcf_score', 50),
            'tech_score': analysis_result.get('tech_score', 50),
            'f_score': analysis_result.get('f_score', 5),

            # 新增因子
            'earnings_revision_score': analysis_result.get('earnings_revision_score', 50),
            'accruals_score': analysis_result.get('accruals_score', 50),

            # 宏觀環境
            'vix': analysis_result.get('vix', 20),
            'yield_curve': analysis_result.get('yield_curve', 0),
            'macro_score': {'risk_on': 2, 'neutral': 0, 'risk_off': -2}.get(
                analysis_result.get('macro_regime', 'neutral'), 0),

            # 行業分類（數字編碼）
            'sector_code': {
                'tech': 0, 'financial': 1, 'healthcare': 2,
                'consumer': 3, 'consumer_staple': 4,
                'cyclical': 5, 'defensive': 6, 'other': 7
            }.get(analysis_result.get('sector_type', 'other'), 7),

            # 因子間的交互特徵
            'quality_x_value': analysis_result.get('quality_score', 50) * analysis_result.get('value_score', 50) / 100,
            'momentum_x_tech': analysis_result.get('momentum_score', 50) * analysis_result.get('tech_score', 50) / 100,
            'analyst_x_insider': analysis_result.get('analyst_score', 50) * analysis_result.get('insider_score', 50) / 100,
        }

        return features

    def train(self, historical_data, labels, n_splits=5):
        """
        使用Walk-Forward交叉驗證訓練集成模型

        參數：
            historical_data: DataFrame，每行是一筆歷史預測的特徵
            labels: 二分類標籤（1=成功, 0=失敗）或回報率
            n_splits: Walk-Forward折數
        """
        self.feature_names = list(historical_data.columns)
        X = historical_data.values
        y = labels.values if hasattr(labels, 'values') else np.array(labels)

        # 時間序列切分（Walk-Forward）
        tscv = TimeSeriesSplit(n_splits=n_splits, gap=5)  # gap=5天隔離帶

        # 第一層模型的超參數
        lgb_params = {
            'objective': 'binary',
            'metric': 'auc',
            'num_leaves': 31,
            'learning_rate': 0.05,
            'feature_fraction': 0.8,
            'bagging_fraction': 0.8,
            'bagging_freq': 5,
            'verbose': -1,
            'n_estimators': 200,
            'reg_alpha': 0.1,
            'reg_lambda': 0.1,
            'min_child_samples': 10,
        }

        xgb_params = {
            'objective': 'binary:logistic',
            'eval_metric': 'auc',
            'max_depth': 6,
            'learning_rate': 0.05,
            'n_estimators': 200,
            'subsample': 0.8,
            'colsample_bytree': 0.8,
            'reg_alpha': 0.1,
            'reg_lambda': 0.1,
            'min_child_weight': 5,
            'verbosity': 0,
        }

        # 收集OOF（out-of-fold）預測用於訓練元模型
        oof_lgb = np.zeros(len(X))
        oof_xgb = np.zeros(len(X))
        oof_ridge = np.zeros(len(X))

        for fold, (train_idx, val_idx) in enumerate(tscv.split(X)):
            X_train, X_val = X[train_idx], X[val_idx]
            y_train, y_val = y[train_idx], y[val_idx]

            # LightGBM
            lgb_model = lgb.LGBMClassifier(**lgb_params)
            lgb_model.fit(X_train, y_train,
                         eval_set=[(X_val, y_val)],
                         callbacks=[lgb.early_stopping(20, verbose=False)])
            oof_lgb[val_idx] = lgb_model.predict_proba(X_val)[:, 1]

            # XGBoost
            xgb_model = xgb.XGBClassifier(**xgb_params)
            xgb_model.fit(X_train, y_train,
                         eval_set=[(X_val, y_val)],
                         verbose=False)
            oof_xgb[val_idx] = xgb_model.predict_proba(X_val)[:, 1]

            # Ridge回歸
            ridge_model = Ridge(alpha=1.0)
            ridge_model.fit(X_train, y_train)
            oof_ridge[val_idx] = ridge_model.predict(X_val)

        # 訓練元模型（簡單線性組合）
        # 只使用有OOF預測的部分
        valid_mask = (oof_lgb != 0) | (oof_xgb != 0) | (oof_ridge != 0)
        meta_X = np.column_stack([oof_lgb[valid_mask], oof_xgb[valid_mask], oof_ridge[valid_mask]])
        meta_y = y[valid_mask]

        self.meta_model = Ridge(alpha=0.5)
        self.meta_model.fit(meta_X, meta_y)

        # 用全部數據重新訓練第一層模型
        self.lgb_model = lgb.LGBMClassifier(**lgb_params)
        self.lgb_model.fit(X, y)

        self.xgb_model = xgb.XGBClassifier(**xgb_params)
        self.xgb_model.fit(X, y)

        self.ridge_model = Ridge(alpha=1.0)
        self.ridge_model.fit(X, y)

        self.is_trained = True

        # 回報OOF績效
        oof_final = self.meta_model.predict(meta_X)
        oof_binary = (oof_final > 0.5).astype(int)
        accuracy = accuracy_score(meta_y, oof_binary)

        return {
            "oof_accuracy": round(accuracy * 100, 1),
            "n_samples": len(X),
            "n_features": X.shape[1],
            "feature_importance": dict(zip(
                self.feature_names,
                self.lgb_model.feature_importances_.tolist()
            )),
        }

    def predict(self, features_dict):
        """
        對新股票進行預測

        回傳：
            dict: 包含預測機率和信心等級
        """
        if not self.is_trained:
            raise ValueError("模型尚未訓練，請先呼叫 train()")

        # 轉換特徵
        X = np.array([[features_dict.get(f, 0) for f in self.feature_names]])

        # 第一層預測
        lgb_pred = self.lgb_model.predict_proba(X)[:, 1][0]
        xgb_pred = self.xgb_model.predict_proba(X)[:, 1][0]
        ridge_pred = self.ridge_model.predict(X)[0]

        # 元模型組合
        meta_X = np.array([[lgb_pred, xgb_pred, ridge_pred]])
        final_prob = float(self.meta_model.predict(meta_X)[0])
        final_prob = max(0, min(1, final_prob))  # 確保在 [0, 1] 範圍

        # 轉換為信心等級和方向
        if final_prob > 0.75:
            direction = "強烈看漲"
            confidence = "高"
        elif final_prob > 0.60:
            direction = "看漲"
            confidence = "中高"
        elif final_prob > 0.52:
            direction = "輕度看漲"
            confidence = "中"
        elif final_prob > 0.48:
            direction = "中性/觀望"
            confidence = "低"
        elif final_prob > 0.35:
            direction = "看跌"
            confidence = "中"
        else:
            direction = "強烈看跌"
            confidence = "高"

        return {
            "probability": round(final_prob, 4),
            "direction": direction,
            "confidence": confidence,
            "model_scores": {
                "lightgbm": round(lgb_pred, 4),
                "xgboost": round(xgb_pred, 4),
                "ridge": round(ridge_pred, 4),
            }
        }
```

### 8.4 新聞情緒信號（P1）

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch
import requests
from datetime import datetime, timedelta

class NewsSentimentSignal:
    """
    新聞情緒信號：使用FinBERT分析金融新聞情緒
    """

    def __init__(self, finnhub_api_key=None, model_name="ProsusAI/finbert"):
        self.finnhub_key = finnhub_api_key
        self.model_name = model_name
        self.tokenizer = None
        self.model = None
        self._loaded = False

    def _load_model(self):
        """延遲載入模型（首次使用時）"""
        if not self._loaded:
            self.tokenizer = AutoTokenizer.from_pretrained(self.model_name)
            self.model = AutoModelForSequenceClassification.from_pretrained(self.model_name)
            self.model.eval()
            self._loaded = True

    def analyze_text(self, text):
        """
        分析單條文本的情緒
        回傳：{'positive': float, 'negative': float, 'neutral': float}
        """
        self._load_model()

        inputs = self.tokenizer(text, return_tensors="pt",
                               truncation=True, max_length=512, padding=True)

        with torch.no_grad():
            outputs = self.model(**inputs)
            probs = torch.nn.functional.softmax(outputs.logits, dim=-1)

        labels = ['positive', 'negative', 'neutral']
        scores = {labels[i]: float(probs[0][i]) for i in range(3)}

        return scores

    def get_news_sentiment_score(self, ticker, days_back=30):
        """
        獲取個股近期新聞的綜合情緒分數 (0-100)

        參數：
            ticker: 股票代碼
            days_back: 回看天數

        回傳：
            (score, reasons)
        """
        reasons = []

        # 從Finnhub獲取新聞
        if not self.finnhub_key:
            return 50, ["無新聞API Key"]

        end_date = datetime.now()
        start_date = end_date - timedelta(days=days_back)

        try:
            url = f"https://finnhub.io/api/v1/company-news"
            params = {
                "symbol": ticker,
                "from": start_date.strftime("%Y-%m-%d"),
                "to": end_date.strftime("%Y-%m-%d"),
                "token": self.finnhub_key
            }
            r = requests.get(url, params=params, timeout=15)
            news = r.json()

            if not news:
                return 50, ["近期無相關新聞"]

            # 分析每條新聞的情緒
            sentiments = []
            for article in news[:50]:  # 最多分析50條
                headline = article.get('headline', '')
                summary = article.get('summary', '')
                text = f"{headline}. {summary}"

                sent = self.analyze_text(text)
                # 計算淨情緒分數：-1（全負面）到 +1（全正面）
                net_sentiment = sent['positive'] - sent['negative']
                sentiments.append(net_sentiment)

            if not sentiments:
                return 50, []

            # 計算綜合指標
            avg_sentiment = sum(sentiments) / len(sentiments)
            recent_sentiment = sum(sentiments[:10]) / min(10, len(sentiments))  # 最近10條

            # 情緒趨勢（最近 vs 整體）
            sentiment_momentum = recent_sentiment - avg_sentiment

            # 轉換為0-100分數
            score = 50 + avg_sentiment * 40  # 映射到 10-90 範圍
            score = max(5, min(95, score))

            # 情緒突變加成
            if sentiment_momentum > 0.2:
                score = min(score + 10, 95)
                reasons.append(f"新聞情緒急劇改善(+{sentiment_momentum:.2f})")
            elif sentiment_momentum < -0.2:
                score = max(score - 10, 5)
                reasons.append(f"新聞情緒急劇惡化({sentiment_momentum:.2f})")

            # 新聞量異常
            if len(news) > 30:
                reasons.append(f"新聞關注度高({len(news)}條/月)")

            # 總結
            pos_count = sum(1 for s in sentiments if s > 0.1)
            neg_count = sum(1 for s in sentiments if s < -0.1)

            if pos_count > neg_count * 2:
                reasons.append(f"正面新聞主導({pos_count}正/{neg_count}負)")
            elif neg_count > pos_count * 2:
                reasons.append(f"負面新聞主導({pos_count}正/{neg_count}負)")

            return round(score, 1), reasons

        except Exception as e:
            return 50, [f"新聞分析異常: {str(e)[:50]}"]

    def get_sentiment_without_model(self, ticker, days_back=30):
        """
        不使用ML模型的簡易情緒分析（基於新聞數量和關鍵詞）
        適合快速部署，無需下載大型模型
        """
        if not self.finnhub_key:
            return 50, []

        end_date = datetime.now()
        start_date = end_date - timedelta(days=days_back)

        try:
            url = f"https://finnhub.io/api/v1/company-news"
            params = {
                "symbol": ticker,
                "from": start_date.strftime("%Y-%m-%d"),
                "to": end_date.strftime("%Y-%m-%d"),
                "token": self.finnhub_key
            }
            r = requests.get(url, params=params, timeout=15)
            news = r.json()

            if not news:
                return 50, []

            # 關鍵詞情緒分析
            positive_words = {'upgrade', 'beat', 'surpass', 'exceed', 'growth', 'bullish',
                            'strong', 'outperform', 'raise', 'buy', 'positive', 'record',
                            'surge', 'jump', 'soar', 'rally', 'boom', 'innovation'}
            negative_words = {'downgrade', 'miss', 'decline', 'cut', 'sell', 'bearish',
                            'weak', 'underperform', 'lower', 'negative', 'loss', 'drop',
                            'fall', 'crash', 'risk', 'lawsuit', 'investigation', 'fraud',
                            'recall', 'layoff', 'restructuring'}

            pos_count = 0
            neg_count = 0
            reasons = []

            for article in news[:50]:
                text = (article.get('headline', '') + ' ' + article.get('summary', '')).lower()
                pos_count += sum(1 for w in positive_words if w in text)
                neg_count += sum(1 for w in negative_words if w in text)

            total = pos_count + neg_count
            if total == 0:
                return 50, []

            pos_ratio = pos_count / total
            score = pos_ratio * 100
            score = max(10, min(90, score))

            if pos_ratio > 0.7:
                reasons.append(f"新聞情緒偏正面({pos_ratio:.0%}正面)")
            elif pos_ratio < 0.3:
                reasons.append(f"新聞情緒偏負面({pos_ratio:.0%}正面)")

            return round(score, 1), reasons

        except Exception:
            return 50, []
```

### 8.5 增強型宏觀環境過濾器（P1）

```python
import yfinance as yf
import pandas as pd
import numpy as np

class EnhancedMacroFilter:
    """
    增強型宏觀環境過濾器
    新增：VIX期限結構、市場寬度、信用利差、Put/Call Ratio
    """

    def __init__(self):
        self._cache = {}

    def get_enhanced_regime(self):
        """
        增強型宏觀環境判斷
        回傳：環境狀態 + 詳細指標 + 建議因子權重
        """
        regime = {
            "state": "neutral",
            "score": 0,
            "indicators": {},
            "details": []
        }

        try:
            # 1. VIX水平
            vix = yf.download("^VIX", period="5d", progress=False)
            if not vix.empty:
                if isinstance(vix.columns, pd.MultiIndex):
                    vix.columns = vix.columns.get_level_values(0)
                vix_val = float(vix['Close'].iloc[-1])
                regime["indicators"]["vix"] = round(vix_val, 1)
                if vix_val < 15:
                    regime["score"] += 3
                elif vix_val < 20:
                    regime["score"] += 1
                elif vix_val > 30:
                    regime["score"] -= 3
                    regime["details"].append(f"VIX高位警告({vix_val:.0f})")
                elif vix_val > 25:
                    regime["score"] -= 1

            # 2. VIX期限結構（VIX vs VIX3M代理）
            # 使用VIX9D和VIX的比值作為代理
            vix9d = yf.download("^VIX9D", period="5d", progress=False)
            if not vix9d.empty and not vix.empty:
                if isinstance(vix9d.columns, pd.MultiIndex):
                    vix9d.columns = vix9d.columns.get_level_values(0)
                ratio = float(vix9d['Close'].iloc[-1]) / vix_val if vix_val > 0 else 1
                regime["indicators"]["vix_term_structure"] = round(ratio, 3)
                if ratio > 1.1:
                    regime["score"] -= 2  # Backwardation = 恐慌
                    regime["details"].append("VIX期限結構倒掛(恐慌)")
                elif ratio < 0.85:
                    regime["score"] += 1  # Contango = 平靜

            # 3. 市場寬度（SPY成分股中有多少在200MA以上）
            spy = yf.download("SPY", period="1y", progress=False)
            if not spy.empty:
                if isinstance(spy.columns, pd.MultiIndex):
                    spy.columns = spy.columns.get_level_values(0)
                spy_close = spy['Close']
                sma50 = spy_close.rolling(50).mean().iloc[-1]
                sma200 = spy_close.rolling(200).mean().iloc[-1]
                current = float(spy_close.iloc[-1])

                if current > sma50 > sma200:
                    regime["score"] += 2
                elif current > sma200:
                    regime["score"] += 1
                elif current < sma50 < sma200:
                    regime["score"] -= 2
                    regime["details"].append("SPY跌破50MA和200MA")
                elif current < sma200:
                    regime["score"] -= 1

            # 4. 信用利差代理（HYG vs TLT表現差異）
            hyg = yf.download("HYG", period="3mo", progress=False)
            tlt = yf.download("TLT", period="3mo", progress=False)
            if not hyg.empty and not tlt.empty:
                if isinstance(hyg.columns, pd.MultiIndex):
                    hyg.columns = hyg.columns.get_level_values(0)
                if isinstance(tlt.columns, pd.MultiIndex):
                    tlt.columns = tlt.columns.get_level_values(0)

                hyg_ret = float(hyg['Close'].iloc[-1] / hyg['Close'].iloc[-20] - 1)
                tlt_ret = float(tlt['Close'].iloc[-1] / tlt['Close'].iloc[-20] - 1)
                spread_change = hyg_ret - tlt_ret
                regime["indicators"]["credit_spread_proxy"] = round(spread_change * 100, 2)

                if spread_change > 0.02:
                    regime["score"] += 1  # 信用利差收窄 = Risk-On
                elif spread_change < -0.02:
                    regime["score"] -= 1  # 信用利差擴大 = Risk-Off
                    regime["details"].append("信用利差擴大")

            # 5. 殖利率曲線
            tnx = yf.download("^TNX", period="5d", progress=False)  # 10Y
            irx = yf.download("^IRX", period="5d", progress=False)  # 3M
            if not tnx.empty and not irx.empty:
                if isinstance(tnx.columns, pd.MultiIndex):
                    tnx.columns = tnx.columns.get_level_values(0)
                if isinstance(irx.columns, pd.MultiIndex):
                    irx.columns = irx.columns.get_level_values(0)
                spread = float(tnx['Close'].iloc[-1]) - float(irx['Close'].iloc[-1])
                regime["indicators"]["yield_curve"] = round(spread, 2)
                if spread > 1.0:
                    regime["score"] += 1
                elif spread < 0:
                    regime["score"] -= 2
                    regime["details"].append("殖利率曲線倒掛")

        except Exception as e:
            regime["details"].append(f"部分指標取得失敗: {str(e)[:50]}")

        # 環境分類（5級制）
        s = regime["score"]
        if s >= 5:
            regime["state"] = "strong_risk_on"
        elif s >= 2:
            regime["state"] = "risk_on"
        elif s >= -1:
            regime["state"] = "neutral"
        elif s >= -4:
            regime["state"] = "risk_off"
        else:
            regime["state"] = "strong_risk_off"

        # 建議因子權重
        regime["recommended_weights"] = self._get_weights(regime["state"])

        return regime

    def _get_weights(self, state):
        """根據5級環境分類給出因子權重"""
        weight_map = {
            "strong_risk_on": {
                "quality": 0.10, "value": 0.10, "momentum": 0.35,
                "analyst": 0.10, "insider": 0.05, "dcf": 0.05,
                "earnings_revision": 0.15, "news_sentiment": 0.10,
            },
            "risk_on": {
                "quality": 0.15, "value": 0.15, "momentum": 0.25,
                "analyst": 0.10, "insider": 0.05, "dcf": 0.10,
                "earnings_revision": 0.10, "news_sentiment": 0.10,
            },
            "neutral": {
                "quality": 0.20, "value": 0.20, "momentum": 0.15,
                "analyst": 0.10, "insider": 0.05, "dcf": 0.10,
                "earnings_revision": 0.10, "news_sentiment": 0.10,
            },
            "risk_off": {
                "quality": 0.30, "value": 0.20, "momentum": 0.05,
                "analyst": 0.10, "insider": 0.10, "dcf": 0.05,
                "earnings_revision": 0.10, "news_sentiment": 0.10,
            },
            "strong_risk_off": {
                "quality": 0.35, "value": 0.25, "momentum": 0.00,
                "analyst": 0.10, "insider": 0.10, "dcf": 0.00,
                "earnings_revision": 0.10, "news_sentiment": 0.10,
            },
        }
        return weight_map.get(state, weight_map["neutral"])
```
